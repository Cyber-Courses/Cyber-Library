# Upplägg för Offensiv Cybersäkerhet

## Nätverk

### Rekognosering och Nätverkskartläggning
Nätverksrekognosering innebär att samla information om ett mål-nätverk för att skapa en karta över dess struktur och enheter. Tekniker inkluderar passiv informationssamling, scanning och aktiv enumeration av nätverkskomponenter. Verktyg som Nmap och Wireshark används för att identifiera öppna portar, tjänster och potentiella sårbarheter. Använd kommandon som:

```bash
nmap -sS -O -v target-ip
```

Detta kommando utför en TCP SYN-skanning och identifierar målets operativsystem.

### Strategier för Lateral Förflyttning
Lateral förflyttning avser tekniker som angripare använder för att förflytta sig genom nätverket efter första åtkomsten. Detta omfattar utnyttjande av sårbarheter i angränsande nätverkssegment och användning av komprometterade autentiseringsuppgifter för att komma åt resurser. Angripare kan använda verktyg som PsExec eller PowerShell för lateral spridning.

### Utnyttja Nätverksprotokoll
Detta inkluderar att identifiera svagheter i nätverksprotokoll som SMB, RDP eller DNS och utnyttja dem för att verkställa otillåtna handlingar. Kunskap om protokollmekanik och anomaliskapelser kan leda till framgångsrikt utnyttjande.

### Man-in-the-Middle (MitM) Attacktekniker
MitM-attacker innebär att avlyssna och ändra kommunikationen mellan två parter utan upptäckt. Teknikerna kan inkludera ARP-spoofing eller DNS-förgiftning. Angripare använder verktyg som Ettercap och MITMf för att fånga och manipulera trafik.

### Analys och Manipulering av Nätverkstrafik
Trafikanalys innebär att övervaka datapaket i nätverket för att identifiera mönster eller anomalier. Angripare kan omdirigera, droppa eller injicera paket för att störa eller analysera kommunikationer. Verktyg som Tcpdump och Scapy används ofta för dessa ändamål.

## Webbapplikation

### Avancerade Injektionsattacker (SQLi, NoSQLi)
SQL-injektion innebär att infoga skadlig SQL-kod i användarinmatningsfält för att manipulera eller extrahera data från en databas. Principen sträcker sig till NoSQL-databaser med NoSQLi. SQLmap är ett vanligt verktyg för att automatisera SQL-injektion och överta databaser:

```bash
sqlmap -u "http://target.com/search?q=test" --dbs
```

Detta kommando försöker extrahera databasanamn från målets URL.

### Cross-Site Scripting (XSS) och CSRF-exploatering
XSS tillåter angripare att injicera skript på webbsidor som andra användare ser, vilket möjliggör sessionkapning eller datastöld. CSRF lurar användare att utföra oönskade åtgärder på en annan webbplats där de är autentiserade.

### Insecure Direct Object Reference (IDOR) och BOLAs
IDOR-sårbarheter tillåter angripare att komma åt data genom att manipulera referenser till objekt (t.ex. användar-ID:n). Broken Object Level Authorization (BOLAs) innebär bristfälligt genomförande av åtkomstkontroll över enskilda objekt.

### Webbskalstaktiker och Persistens
Installera ett webbskal på en komprometterad server för ihållande åtkomst och fjärrkommandoexekvering. Webbskal är enkla skript som tillhandahåller ett gränssnitt via HTTP.

```php
<?php system($_GET['cmd']); ?>
```

Exempel på ett enkelt PHP-webbskal.

### Undvikande av Web Application Firewall
Undvikande av WAF:er innebär att skapa nyttolaster som kringgår signaturbaserad detektion. Teknikerna inkluderar kodning, obfuskering eller fragmentering av attackvektorer.

## Mobilapplikation

### Omvänd Ingenjörskonst av Mobilapplikationer
Omvänd ingenjörskonst innebär att dekryptera och analysera mobilappskod för att förstå dess logik och identifiera sårbarheter. Verktyg som APKTool eller Hopper är oumbärliga för disassembly eller dekompilering.

### Exploatering av Sårbarheter på Mobilplattformar
Varje mobilplattform (t.ex. Android, iOS) har specifika sårbarheter. Exploateringar kan innebära att utnyttja systemprivilegier eller föråldrade bibliotek.

### Avlyssning och Manipulering av Mobilkommunikation
Angripare använder verktyg som Burp Suite eller Frida för att avlyssna och manipulera data som överförs av mobilapplikationer, särskilt när SSL/TLS-implementeringar är svaga eller felkonfigurerade.

### Dataintrång i Mobilapplikationer
Attackera osäkra datalagrings- eller dataöverföringsmetoder som används av mobilappar där känslig information lagras eller kommuniceras utan tillräcklig kryptering.

## Leveranskedja

### Attack mot Utvecklings- och Leveransflöden
Kompromettering av utvecklingsflödet kan leda till en omfattande spridning av skadlig kod. Angripare riktar in sig på källkodsarkiv eller CI/CD-miljöer för att injicera bakdörrar.

### Säkerhetstestning av Inbäddade System
Fokusera på att identifiera sårbarheter inom inbyggd enhets firmware eller programvara. Testning omfattar ofta hårdvarugränssnitt som JTAG eller UART.

### Tekniker för Kompromettering av Leveranskedjan
Infoga skadliga komponenter inom en produkts leveranskedja för att störa integriteten innan det når slutanvändaren. Noterbara exempel inkluderar hårdvaruimplanteringar eller manipulerade uppdateringsfiler.

### Bakdörrning av Programvara från Tredje Part
Införande av dold, otillåten kod i programvaruprodukter levererade av tredjepartsleverantörer under utveckling, antingen avsiktligt eller via exploatering.

## CI/CD

### Kompromettering av CI/CD-miljöer
Otillåten åtkomst till en CI/CD-miljö ger makten att förändra mjukvarubyggprocessen, potentiellt injicera skadlig kod före distribution.

### Infiltrera Byggpipeline
Angripare fångar upp eller modifierar byggartefakter, vilket leder till distribution av komprometterad programvara. Sårbarheter i byggskript eller beroenden är vanligtvis utnyttjade.

### Manipulering av Artefakter och Integritetskompromettering
Modifikation av byggartefakter utan upptäckt kan underminera mjukvarans avsedda funktionalitet. Att säkerställa artefaktsintegritet är avgörande och förlitar sig på checksummor eller kryptografiska signaturer.

### Exploatering av Sårbarheter i CI/CD-verktyg
Kända sårbarheter i CI/CD-verktyg som Jenkins eller GitLab kan utnyttjas för otillåten åtkomst och kodändring.

## Moln

### Exploateringstekniker för Molntjänster
Utnyttja misskonfigurationer eller sårbarheter i molntjänster för att få otillåten åtkomst eller kontroll över molnhanterade resurser.

### Misskonfigurerade Molnlagringsattacker
Angripa exponerad lagringstjänster som AWS S3-ämnen på grund av otillräckliga säkerhetskontroller som offentlig skriv- eller läsåtkomst.

### Förbikoppling av Identitets- och Åtkomsthantering i Molnet
Få otillåten åtkomst till molntillgångar genom att utnyttja svaga IAM-policys eller överdrivet tillåtande roller.

### Attackvektorer för Serverlös Arkitektur
Utnyttja sårbarheter i serverlösa ramverk genom att missbruka händelsesor triggare eller injicera skadliga nyttolaster i funktionens verkställandekontext.

## Container

### Tekniker för Containerflykt
Identifiera och utnyttja sårbarheter som tillåter skadlig kod att fly ut från containermiljön till värdsystemet.

### Attackera Kubernetes-kluster
Utnyttja misskonfigurationer inom Kubernetes-miljöer för att få otillåten kontroll eller störa tjänst.

### Exploateering av Sårbarheter i Containerorkestrering
Riktade angrepp på sårbarheter i orkestreringsverktyg som Docker Swarm eller Kubernetes för att få tillgång till begränsade miljöer.

### Förgiftning av Containerbilder
Manipulera containerbilder genom att injicera skadlig kod, som påverkar system där dessa bilder distribueras.

## API

### Enumeration och Rekognosering av API Endpoints
Identifiera API-slutpunkter och kartlägga deras struktur för att bestämma tillgängliga resurser och potentiella sårbarheter.

### Förbikoppling av Autentisering och Auktorisation för API
Utnyttja sårbarheter som tillåter förbikoppling av autentiseringslager eller bristfälliga auktorisationskontroller.

### Missbruk av Hastighetsbegränsning och Kvot
Förbikoppla hastighetsbegränsningar för att överväldiga API:et eller extrahera mer data än avsett genom att manipulera förfrågningshuvuden eller nyttolaster.

### Exploatering av Sårbarheter i REST och SOAP API
Rikta in sig på specifika API:er genom injektion, deserialiseringsattacker eller genom att utnyttja exponering på grund av detaljerade felmeddelanden.

## Hårdvara

### Exploatering av Hårdvarugränssnitt
Manipulera eller komma åt hårdvarugränssnitt för att extrahera känslig data eller ändra enheters operationer.

### Sidokanalattacker på Hårdvara
Utnyttja elektromagnetiska emissioner, strömförbrukningsanalys eller tidsinformation för att förstå enheters operationer eller hämta känslig information.

### Intrång i Inbyggda Enheter
Få otillåten åtkomst till inbäddade enheter genom sårbarheter i firmware eller externa kommunikationsgränssnitt.

### Omvänd Ingenjörskonst av Hårdvarufirmware
Analysera firmware för hårdvaruenheter för att identifiera sårbarheter eller förstå enhetens operationer genom omvänd ingenjörskonst.

## Trådlöst

### Trådlösa Nätverksattacker (Wi-Fi, Bluetooth)
Utnyttja svaga Wi-Fi-protokoll (WEP/WPA2) eller Bluetooth-misskonfigurationer för att avlyssna och manipulera trådlösa kommunikationer.

### Utplacering av Bedrägliga Accesspunkter
Att upprätta obehöriga accesspunkter för att avlyssna trådlösa kommunikationer, en teknik som ofta används vid nätfiske eller stöld av auktorisationsuppgifter.

### Avlyssning av Trådlösa Kommunikationer
Avlyssna okrypterad kommunikation över ett trådlöst nätverk för att fånga känslig information såsom lösenord.

### Exploatering av Trådlösa Protokoll
Utnyttja svagheter i protokoll som Zigbee eller Bluetooth Low Energy (BLE) för att få kontroll eller avlyssna enhetskommunikationer.

## Fysiska Säkerhetsattacker

### Social Engineering och Fysisk Penetration
Använda bedrägeri för att kringgå fysiska säkerhetsåtgärder, vilket inkluderar att imitera eller förfalska legitimationer för att få obehörig åtkomst till anläggningar.

### Förbikoppling av Fysiska Åtkomstkontroller
Övervinna barriärer som lås eller biometriska system genom tekniker som dyrkning eller reläattacker.

### Exfiltrering av Information på den Fysiska Lagret
Extrahera data genom fysiska medel, såsom kopiering av data från exponerade hårddiskar eller avlyssning av elektromagnetiska signaler.

### Metoder för Bedömning av Fysisk Säkerhet
Simuleringsbaserade tekniker för att bedöma sårbarheter i fysiska säkerhetsinställningar, vilket innebär penetrationstestning av lokaler.

## Kryptografisk

### Angrepp på Kryptografiska Protokoll
Utnyttja sårbarheter i kryptografiska protokoll, vilket leder till datadekryptering, autentiseringsförbikoppling eller imitation.

### Kryptanalystekniker
Tekniker som använder matematisk och systemanalys för att kompromettera kryptografisk säkerhet, inklusive bryta chiffer.

### Missbruk av Nyckelhantering
Attacker som utnyttjar dåliga nyckelhanteringspraxis leder till otillåten nyckelavslöjande och datadekryptering.

### Brytning av Symmetrisk och Asymmetrisk Kryptering
Hitta svagheter eller utnyttja implementationsfel för att dekryptera data utan nyckelinnehav. Exempel inkluderar attacker mot AES eller RSA.

## Exploitutveckling

### Identifiera Möjligheter för Sårbarhetsexploatering
Forska och upptäck sårbarheter i mjukvara eller hårdvara som kan utnyttjas. Fokus på ingångspunkter inom okontrollerade indata eller okontrollerade operationer.

### Avancerade Minneskorrumptionstekniker
Utnyttja svagheter i minnesoperationer för att uppnå godtycklig kodexekvering, inklusive bufferoverflows och use-after-free-exploater.

### Utformning och Leverans av Shellcode
Designa och injicera shellcode som en nyttolast för att exekvera skadliga instruktioner på ett målsystem.

### Exploatramverksanvändning
Använda verktyg som Metasploit för att underlätta exploitutveckling och hantera attackvektorer genom konfigurerbara moduler för varierande mål.

## Red Teaming

### Planering och Utförande av Red Team Engagemang
Red teaming innebär simulering av avancerade ihållande hot (APT:er) för att utvärdera organisationers försvar. Planering fokuserar på att uppnå specifika taktiker, tekniker och procedurer (TTP:er).

### Setup av Infrastruktur och Verktyg för Red Team
Bygga en säker och motståndskraftig infrastruktur för att genomföra red team-operationer, med fokus på kommandokontroll (C2) system och attackverktyg.

### Flerstegsangreppskampanjer
Koordinerade kampanjer som efterliknar verkliga angripare, inbegriper flera faser som rekognosering, exploatering och lateral förflyttning.

### Simulering och Imitation av Motståndare
Realistisk simulering av taktik som används av avancerade hotaktörer för att utvärdera en organisations försvarsåtgärder och identifiera förbättringsområden.

## Undvikande

### Förbikoppling av Nätverks- och Slutpunktsskydd
Använda taktiker för att undvika detektion av IDS/IPS och slutpunktsskyddslösningar genom malware-obfuskering eller utnyttja detektionsluckor.

### Anti-Forensik och Manipulering av Bevis
Tekniker utformade för att förhindra forensisk analys eller utplåna bevisningstråk, kritiska för upprätthållande av hemliga operationer.

### Tekniker för Obfuskering och Kryptering
Maskera nyttolaster eller kommunikationer genom lager av obfuskering eller kryptering för att undvika säkerhetsåtgärder och analys.

### Undvikande av Honungsburkar och Sandlådor
Upptäcka och undvika interaktion med locksystem som honungsburkar eller sandlådemiljöer som syftar till att fånga skadlig aktivitet.

## Malwareutveckling

### Design och Leverans av Användarspecifika Payloads
Skapa skräddarsydda nyttolaster, ofta riktade mot specifika miljöer eller kringgång av befintliga försvar, med betoning på smygande och funktionalitet.

### Tekniker för Undvikelse och Persistens för Malware
Utforma malware för att bestå på systemet efter kompromiss och undvika detektion genom obfuskering eller manipulation av värdförsvar.

### Utveckling och Hantering av C2 Infrastruktur
Bygga och hantera kommando- och kontrollinfrastruktur för att bibehålla autonomi över komprometterade system, nyckel för ihållande hotaktörer.

### Metoder för Att Förbigå Anti-Malware
Ansträngningar för att kringgå automatiserade malwaredetektionssystem med undvikningstekniker som signaturmanipulation eller beteendemodifikation.

## Programvara

### Tekniker för Bakdörrning av Programvara
Införa dolda åtkomstpunkter i programvarukod för obehörig åtkomst, ofta via manipulerade open source- eller proprietära kodbaser.

### Manipulering av ApplikationsLogik
Utnyttja fel i affärslogik eller applikationsflöde för att verkställa oavsiktliga handlingar eller felaktigt extrahera data.

### Avancerade Omvänd Ingenjörstekniker
Djupomfattande analys av programexekverbara filer med verktyg som IDA Pro eller Ghidra för att avslöja intern logik eller sårbarheter i sluten källkodssystem.

### Injicering av Kod och Hookning
Injicera skadlig kod i körande processer eller avlyssna applikationsarbetsflöden genom enleveringstekniker, ofta används för privilegieförhöjning.
