# Roteiro de Cibersegurança Ofensiva

## Rede

### Reconhecimento e Mapeamento de Rede
O reconhecimento de rede envolve a coleta de informações sobre uma rede-alvo para criar um mapa de sua estrutura e dispositivos. As técnicas incluem coleta de informações passiva, varredura e enumeração ativa de componentes de rede. Ferramentas como Nmap e Wireshark são utilizadas para identificar portas abertas, serviços e potenciais vulnerabilidades. Use comandos como:

```bash
nmap -sS -O -v target-ip
```

Este comando realiza uma varredura TCP SYN e detecta o sistema operacional do alvo.

### Estratégias de Movimento Lateral
Movimento lateral refere-se a técnicas que os atacantes usam para mover-se pela rede após o acesso inicial. Isso inclui explorar vulnerabilidades em segmentos de rede adjacentes e usar credenciais comprometidas para acessar recursos. Ferramentas como PsExec ou PowerShell podem ser utilizadas para propagação lateral.

### Exploração de Protocolos de Rede
Isso envolve a identificação de fraquezas em protocolos de rede como SMB, RDP ou DNS e o uso delas para executar ações não autorizadas. O conhecimento dos mecanismos dos protocolos e a criação de anomalias podem levar a uma exploração bem-sucedida.

### Técnicas de Ataque Man-in-the-Middle (MitM)
Os ataques MitM envolvem a interceptação e alteração da comunicação entre duas partes sem serem detectados. As técnicas podem incluir ARP spoofing ou envenenamento DNS. Os atacantes utilizam ferramentas como Ettercap e MITMf para capturar e manipular o tráfego.

### Análise e Manipulação de Tráfego de Rede
A análise de tráfego envolve monitorar pacotes de dados de rede para identificar padrões ou anomalias. Os atacantes podem redirecionar, descartar ou injetar pacotes para interromper ou dissecar as comunicações. Ferramentas como Tcpdump e Scapy são comumente usadas para esses fins.

## Aplicação Web

### Ataques Avançados de Injeção (SQLi, NoSQLi)
A Injeção SQL envolve a inserção de código SQL malicioso em campos de entrada para manipular ou extrair dados de um banco de dados. O princípio se estende a bancos de dados NoSQL com NoSQLi. SQLmap é uma ferramenta comum para automatizar injeções SQL e tomar controle de banco de dados:

```bash
sqlmap -u "http://target.com/search?q=test" --dbs
```

Este comando tenta extrair nomes de bancos de dados a partir da URL alvo.

### Exploração de Cross-Site Scripting (XSS) e CSRF
XSS permite que atacantes injetem scripts em páginas web vistas por outros usuários, possibilitando sequestro de sessões ou roubo de dados. CSRF engana usuários para executar ações indesejadas em um site diferente onde estão autenticados.

### Referência Direta de Objeto Insegura (IDOR) e BOLAs
Vulnerabilidades IDOR permitem que atacantes acessem dados manipulando referências a objetos (por exemplo, IDs de usuários). A Autorização de Nível de Objeto Quebrada (BOLAs) envolve a aplicação inadequada do controle de acesso sobre objetos individuais.

### Táticas de Web Shell e Persistência
Instalar uma web shell em um servidor comprometido para acesso persistente e execução remota de comandos. Web shells são scripts simples que fornecem uma interface via HTTP.

```php
<?php system($_GET['cmd']); ?>
```

Exemplo de uma web shell PHP simplista.

### Evasão de Firewall de Aplicação Web
Evasão de WAFs envolve a criação de payloads que contornem a detecção baseada em assinaturas. As técnicas incluem codificação, ofuscação ou fragmentação de vetores de ataque.

## Aplicação Móvel

### Engenharia Reversa de Aplicações Móveis
Engenharia reversa envolve a descriptografia e análise do código de um aplicativo móvel para entender sua lógica e identificar vulnerabilidades. Ferramentas como APKTool ou Hopper são essenciais para desmontagem ou descompilação.

### Exploração de Vulnerabilidades em Plataformas Móveis
Cada plataforma móvel (por exemplo, Android, iOS) tem vulnerabilidades específicas. As explorações podem envolver o uso de privilégios do sistema ou bibliotecas desatualizadas.

### Interceptação e Manipulação de Comunicações Móveis
Os atacantes usam ferramentas como Burp Suite ou Frida para interceptar e manipular dados transmitidos por aplicativos móveis, especialmente quando implementações SSL/TLS são fracas ou configuradas incorretamente.

### Quebras de Segurança de Dados de Aplicativos Móveis
Alvo de armazenamento de dados ou métodos de transmissão inseguros usados por aplicativos móveis, onde informações sensíveis são armazenadas ou comunicadas sem criptografia adequada.

## Cadeia de Abastecimento

### Alvo no Desenvolvimento e Entrega de Pipelines
Comprometer a linha de desenvolvimento pode levar à distribuição generalizada de código malicioso. Os atacantes miram repositórios de código-fonte ou ambientes CI/CD para injetar backdoors.

### Teste de Segurança de Sistemas Embutidos
Foco na identificação de vulnerabilidades dentro do firmware ou software de dispositivos embutidos. O teste muitas vezes inclui interfaces de hardware como JTAG ou UART.

### Técnicas de Comprometimento da Cadeia de Abastecimento
Inserir componentes maliciosos dentro da cadeia de abastecimento de um produto para interromper a integridade antes de ele chegar ao usuário final. Instâncias notáveis incluem implantes de hardware ou arquivos de atualização adulterados.

### Backdooring de Software de Terceiros
Introdução de código não autorizado e oculto em produtos de software fornecidos por fornecedores terceiros durante o desenvolvimento, intencionalmente ou via exploração.

## CI/CD

### Comprometimento de Ambientes CI/CD
O acesso não autorizado a um ambiente CI/CD concede o poder de alterar o processo de construção de software, potencialmente inserindo código malicioso antes da implantação.

### Infiltração de Pipelines de Construção
Os atacantes interceptam ou modificam artefatos de construção, levando à distribuição de software comprometido. Vulnerabilidades em scripts de construção ou dependências são comumente exploradas.

### Falsificação de Artefatos e Comprometimento de Integridade
A modificação de artefatos de construção não detectada pode subverter a funcionalidade pretendida do software. Garantir a integridade dos artefatos é crucial, confiando em somatórios ou assinaturas criptográficas.

### Exploração de Vulnerabilidades de Ferramentas CI/CD
Vulnerabilidades conhecidas em ferramentas CI/CD, como Jenkins ou GitLab, podem ser exploradas para acesso não autorizado e alteração de código.
