# Hoja de Ruta de Ciberseguridad Ofensiva

## Red

### Reconocimiento y Mapeo de Red
El reconocimiento de red implica recopilar información sobre una red objetivo para crear un mapa de su estructura y dispositivos. Las técnicas incluyen recopilación de información pasiva, escaneo y enumeración activa de componentes de la red. Se utilizan herramientas como Nmap y Wireshark para identificar puertos abiertos, servicios y vulnerabilidades potenciales. Utiliza comandos como:

```bash
nmap -sS -O -v target-ip
```

Este comando realiza un escaneo TCP SYN y detecta el sistema operativo objetivo.

### Estrategias de Movimiento Lateral
El movimiento lateral se refiere a técnicas que los atacantes usan para pivotar a través de la red después del acceso inicial. Esto incluye explotar vulnerabilidades en segmentos de red adyacentes y usar credenciales comprometidas para acceder a recursos. Los atacantes pueden utilizar herramientas como PsExec o PowerShell para la propagación lateral.

### Explotación de Protocolos de Red
Esto implica identificar debilidades en protocolos de red como SMB, RDP o DNS y aprovecharlas para ejecutar acciones no autorizadas. El conocimiento de la mecánica de los protocolos y la creación de anomalías puede conducir a una explotación exitosa.

### Técnicas de Ataque Man-in-the-Middle (MitM)
Los ataques MitM implican interceptar y alterar la comunicación entre dos partes sin ser detectados. Las técnicas pueden incluir suplantación de ARP o envenenamiento de DNS. Los atacantes utilizan herramientas como Ettercap y MITMf para capturar y manipular el tráfico.

### Análisis y Manipulación de Tráfico de Red
El análisis de tráfico implica monitorear los paquetes de datos de la red para identificar patrones o anomalías. Los atacantes pueden redirigir, eliminar o inyectar paquetes para interrumpir o disectar comunicaciones. Herramientas como Tcpdump y Scapy son comúnmente utilizadas para estos propósitos.

## Aplicación Web

### Ataques de Inyección Avanzados (SQLi, NoSQLi)
La inyección SQL implica insertar código SQL malicioso en campos de entrada de usuario para manipular o extraer datos de una base de datos. El principio se extiende a bases de datos NoSQL con NoSQLi. SQLmap es una herramienta común para automatizar la inyección SQL y la toma de control de bases de datos:

```bash
sqlmap -u "http://target.com/search?q=test" --dbs
```

Este comando intenta extraer nombres de bases de datos de la URL objetivo.

### Explotación de Cross-Site Scripting (XSS) y CSRF
XSS permite a los atacantes inyectar scripts en páginas web vistas por otros usuarios, habilitando el secuestro de sesiones o el robo de datos. CSRF engaña a los usuarios para que ejecuten acciones no deseadas en otro sitio donde están autenticados.

### Referencia Directa Insegura a Objetos (IDOR) y BOLAs
Las vulnerabilidades de IDOR permiten a los atacantes acceder a datos manipulando referencias a objetos (por ejemplo, IDs de usuario). Las autorizaciones rotas a nivel de objeto (BOLAs) involucran una aplicación inadecuada del control de acceso sobre objetos individuales.

### Tácticas de Web Shell y Persistencia
Instala un web shell en un servidor comprometido para acceso persistente y ejecución remota de comandos. Los web shells son scripts simples que proporcionan una interfaz a través de HTTP.

```php
<?php system($_GET['cmd']); ?>
```

Ejemplo de un web shell en PHP simplista.

### Evasión de Cortafuegos de Aplicaciones Web
Evitar WAFs implica crear cargas útiles que evaden la detección basada en firmas. Las técnicas incluyen codificación, ofuscación o fragmentación de vectores de ataque.

## Aplicación Móvil

### Ingeniería Inversa de Aplicaciones Móviles
La ingeniería inversa implica descomprimir y analizar el código de una aplicación móvil para entender su lógica e identificar vulnerabilidades. Herramientas como APKTool o Hopper son esenciales para el desensamblaje o descompilación.

### Explotación de Vulnerabilidades en Plataformas Móviles
Cada plataforma móvil (por ejemplo, Android, iOS) tiene vulnerabilidades específicas. Los exploits pueden implicar el uso de privilegios del sistema o bibliotecas desactualizadas.

### Interceptación y Manipulación de Comunicaciones Móviles
Los atacantes utilizan herramientas como Burp Suite o Frida para interceptar y manipular datos transmitidos por aplicaciones móviles, especialmente cuando las implementaciones SSL/TLS son débiles o mal configuradas.

### Brechas de Seguridad en Datos de Aplicaciones Móviles
Apuntando a métodos de almacenamiento o transmisión de datos inseguros utilizados por aplicaciones móviles, donde la información sensible se almacena o comunica sin el cifrado adecuado.

## Cadena de Suministro

### Objetivos en las Pipeplines de Desarrollo y Entrega
Comprometer la pipeline de desarrollo puede llevar a la distribución masiva de código malicioso. Los atacantes apuntan a repositorios de código fuente o entornos CI/CD para inyectar puertas traseras.

### Pruebas de Seguridad en Sistemas Embebidos
Enfocarse en la identificación de vulnerabilidades dentro del firmware o software de dispositivos embebidos. Las pruebas a menudo incluyen interfaces de hardware como JTAG o UART.

### Técnicas de Compromiso de la Cadena de Suministro
Insertar componentes maliciosos dentro de la cadena de suministro de un producto interrumpe la integridad antes de que llegue al usuario final. Instancias notables incluyen implantes de hardware o archivos de actualización manipulados.

### Puertas Traseras en Software de Terceros
Introducir código no autorizado oculto en productos de software suministrados por vendedores de terceros durante el desarrollo, intencionadamente o vía exploit.

## CI/CD

### Comprometiendo Entornos CI/CD
El acceso no autorizado a un entorno CI/CD otorga poder para alterar el proceso de construcción de software, insertando potencialmente código malicioso antes de la implementación.

### Infiltración en Pipelines de Construcción
Los atacantes interceptan o modifican artefactos de construcción, lo que lleva a la distribución de software comprometido. Las vulnerabilidades en scripts de construcción o dependencias son comúnmente explotadas.

### Manipulación e Integridad de Artefactos
La modificación de artefactos de construcción sin detección puede subvertir la funcionalidad prevista del software. Asegurar la integridad de los artefactos es crucial, confiando en checksums o firmas criptográficas.

### Explotación de Vulnerabilidades en Herramientas CI/CD
Las vulnerabilidades conocidas en herramientas CI/CD, como Jenkins o GitLab, pueden ser explotadas para acceso no autorizado y alteración de código.

## Nube

### Técnicas de Explotación de Servicios en la Nube
Explotar configuraciones incorrectas o vulnerabilidades en servicios en la nube para obtener acceso no autorizado o control sobre recursos administrados en la nube.

### Ataques por Configuraciones Incorrectas en Almacenamiento en la Nube
Apuntar a servicios de almacenamiento expuestos como AWS S3 debido a controles de seguridad inadecuados, como acceso de escritura o lectura públicos.

### Evasión de Identidad y Gestión de Acceso en la Nube
Obtener acceso no autorizado a activos en la nube al explotar políticas IAM débiles o roles excesivamente permisivos.

### Vectores de Ataque en Arquitectura Serverless
Explotar vulnerabilidades en frameworks serverless abusando de activadores de eventos o inyectando cargas maliciosas en contextos de ejecución de funciones.

## Contenedores

### Técnicas de Escape de Contenedores
Identificar y explotar vulnerabilidades que permiten a código malicioso escapar del entorno del contenedor hacia el sistema anfitrión.

### Ataques a Clústeres de Kubernetes
Explotar configuraciones incorrectas dentro de los entornos de Kubernetes para obtener control no autorizado o interrumpir el servicio.

### Explotación de Vulnerabilidades en Orquestación de Contenedores
Apuntar a vulnerabilidades en herramientas de orquestación como Docker Swarm o Kubernetes para acceder a entornos restringidos.

### Envenenamiento de Imágenes de Contenedores
Manipular imágenes de contenedores inyectando código malicioso, afectando sistemas donde estas imágenes son desplegadas.

## API

### Enumeración y Reconocimiento de Endpoints de API
Identificar endpoints de API y mapear su estructura para determinar recursos disponibles y vulnerabilidades potenciales.

### Bypass de Autenticación y Autorización de API
Explotar vulnerabilidades que permiten eludir capas de autenticación o comprobaciones de autorización inadecuadas.

### Abuso de Límite de Frecuencia y Cuotas
Eludir los límites de frecuencia para sobrecargar la API o extraer más datos de los previstos manipulando encabezados de solicitud o cargas.

### Explotación de Vulnerabilidades en API REST y SOAP
Apuntar a APIs específicas a través de inyección, ataques de deserialización, o aprovecharse de la exposición debido a mensajes de error detallados.

## Hardware

### Explotación de Interfaces de Hardware
Manipular o acceder a interfaces de hardware para extraer datos sensibles o modificar operaciones del dispositivo.

### Ataques de Canal Lateral en Hardware
Aprovechar emisiones electromagnéticas, análisis de consumo de energía o información de tiempo para inferir operaciones de dispositivos o recuperar información sensible.

### Brecha en Dispositivos Embebidos
Obtener acceso no autorizado a dispositivos embebidos a través de vulnerabilidades en firmware o interfaces de comunicación externas.

### Ingeniería Inversa de Firmware de Hardware
Analizar el firmware de dispositivos de hardware para identificar vulnerabilidades o entender la operación del dispositivo a través de la ingeniería inversa.

## Inalámbrico

### Ataques a Redes Inalámbricas (Wi-Fi, Bluetooth)
Explotar protocolos Wi-Fi débiles (WEP/WPA2) o configuraciones incorrectas de Bluetooth para interceptar y manipular comunicaciones inalámbricas.

### Despliegue de Puntos de Acceso Falsos
Desplegar puntos de acceso no autorizados para interceptar comunicaciones inalámbricas, una técnica a menudo utilizada en phishing o robo de credenciales.

### Espionaje de Comunicaciones Inalámbricas
Escuchar comunicaciones no cifradas sobre una red inalámbrica para capturar información sensible como contraseñas.

### Explotación de Protocolos Inalámbricos
Explotar debilidades en protocolos como Zigbee o Bluetooth Low Energy (BLE) para obtener control o espiar comunicaciones de dispositivos.

## Ataques de Seguridad Física

### Ingeniería Social y Penetración Física
Usar el engaño para eludir medidas de seguridad física, involucrando suplantación de identidad o falsificación de credenciales para obtener acceso no autorizado a instalaciones.

### Elusión de Controles de Acceso Físico
Burlar barreras como cerraduras o sistemas biométricos a través de técnicas como ganzúas o ataques de retransmisión.

### Exfiltración de Información a Nivel Físico
Extraer datos mediante medios físicos, como copiar datos de discos duros expuestos o interceptar señales electromagnéticas.

### Métodos de Evaluación de Seguridad Física
Técnicas basadas en simulaciones para evaluar vulnerabilidades en configuraciones de seguridad física, involucrando pruebas de penetración de instalaciones.

## Criptográfico

### Ataques a Protocolos Criptográficos
Explotar vulnerabilidades en protocolos criptográficos, lo que lleva a la descifrado de datos, evasión de autenticación o suplantación.

### Técnicas de Criptoanálisis
Las técnicas emplean análisis matemático y sistémico para comprometer la seguridad criptográfica, incluyendo el rompimiento de cifrados.

### Abuso en la Gestión de Claves
Atacar prácticas inadecuadas de gestión de claves que conducen a la divulgación no autorizada de claves y descifrado de datos.

### Rompimiento de Cifrado Simétrico y Asimétrico
Encontrar debilidades o explotar fallas de implementación para descifrar datos sin poseer la clave. Ejemplos incluyen ataques a AES o RSA.

## Desarrollo de Exploits

### Identificación de Oportunidades de Explotación de Vulnerabilidades
Investigar y descubrir vulnerabilidades en software o hardware que pueden ser explotadas. Enfocarse en puntos de entrada dentro de inputs no saneados u operaciones no verificadas.

### Técnicas Avanzadas de Corrupción de Memoria
Aprovechar debilidades en operaciones de memoria para lograr ejecución arbitraria de código, incluyendo desbordamientos de buffer y exploits de use-after-free.

### Diseño y Entrega de Shellcode
Diseñar e inyectar shellcode como una carga útil para ejecutar instrucciones maliciosas en un sistema objetivo.

### Utilización de Marcos de Explotación
Usar herramientas como Metasploit para ayudar en el desarrollo de exploits y gestionar vectores de ataque a través de módulos configurables para objetivos variados.

## Red Teaming

### Planificación y Ejecución de Compromisos de Red Team
El red teaming implica simular amenazas persistentes avanzadas (APT) para evaluar defensas organizacionales. La planificación se enfoca en lograr tácticas, técnicas y procedimientos (TTP) específicos.

### Configuración de Infraestructura y Herramientas de Red Team
Construir una infraestructura segura y resistente para realizar operaciones de red team, enfocándose en sistemas de mando y control (C2) y herramientas de ataque.

### Campañas de Ataque en Múltiples Etapas
Campañas coordinadas que imitan atacantes del mundo real, involucrando múltiples fases como reconocimiento, explotación y movimiento lateral.

### Simulación y Emulación de Adversarios
Simulación realista de tácticas usadas por actores de amenazas avanzadas para evaluar medidas defensivas de una organización e identificar áreas de mejora.

## Evasión

### Elusión de Detección en Red y Endpoint
Utilización de tácticas para evitar la detección por IDS/IPS y soluciones de seguridad para endpoints, a través de la ofuscación de malware o explotando brechas de detección.

### Anti-Forense y Manipulación de Evidencias
Técnicas diseñadas para frustrar el análisis forense o erosionar el rastro de evidencia, críticas para mantener operaciones encubiertas.

### Técnicas de Ofuscación y Cifrado
Enmascarar cargas útiles o comunicaciones a través de capas de ofuscación o cifrado para evadir medidas de seguridad y análisis.

### Evasión de Honeypots y Sandboxes
Detectar y evitar la interacción con sistemas señuelo como honeypots o entornos sandbox que buscan capturar actividades maliciosas.

## Desarrollo de Malware

### Diseño y Entrega de Cargas Útiles Personalizadas
Crear cargas útiles personalizadas, a menudo dirigidas a entornos específicos o eludiendo defensas existentes, enfatiza el sigilo y la funcionalidad.

### Técnicas de Evasión y Persistencia de Malware
Crear malware para persistir en el sistema después de un compromiso y evitar la detección mediante la ofuscación o manipulación de defensas del host.

### Desarrollo y Gestión de Infraestructura C2
Construir y gestionar infraestructura de mando y control para mantener autonomía sobre sistemas comprometidos, clave para actores de amenazas persistentes.

### Métodos de Evasión Anti-Malware
Esfuerzos para eludir sistemas de detección automática de malware usando técnicas de evasión como manipulación de firmas o modificación de comportamiento.

## Software

### Técnicas de Puertas Traseras en Software
Introducir puntos de acceso ocultos en el código de software para acceso no autorizado, a menudo a través de código abierto manipulado o bases de código propietario.

### Manipulación de Lógica de Aplicaciones
Explotar errores en la lógica de negocio o flujo de aplicación para ejecutar acciones no intencionadas o extraer datos inadecuadamente.

### Tácticas Avanzadas de Ingeniería Inversa
Análisis profundo de ejecutables de software usando herramientas como IDA Pro o Ghidra para exponer lógica interna o vulnerabilidades en sistemas de código cerrado.

### Inyección de Código y Hooking
Inyectar código malicioso en procesos en ejecución o interceptando flujos de trabajo de aplicaciones a través de técnicas de hooking, utilizadas a menudo para escalada de privilegios.
