# แผนที่การโจมตีทางไซเบอร์เชิงรุก

## เครือข่าย

### การสำรวจและการสร้างแผนที่เครือข่าย
การสำรวจเครือข่ายเกี่ยวข้องกับการรวบรวมข้อมูลเกี่ยวกับเป้าหมายเครือข่ายเพื่อตั้งแผนผังโครงสร้างและอุปกรณ์ของมัน เทคนิคที่ใช้ได้แก่ การรวบรวมข้อมูลแบบพาสซีฟ การสแกน และการระบุองค์ประกอบของเครือข่ายแบบแอ็คทีฟ เครื่องมือเช่น Nmap และ Wireshark ถูกนำมาใช้ระบุช่องที่เปิดอยู่ บริการที่เปิดใช้งาน และจุดอ่อนที่เป็นไปได้ ใช้คำสั่งเช่น:

```bash
nmap -sS -O -v target-ip
```

คำสั่งนี้จะทำการสแกน TCP SYN และตรวจดูระบบปฏิบัติการของเป้าหมาย

### กลยุทธ์การเคลื่อนที่ด้านข้าง
การเคลื่อนที่ด้านข้างหมายถึงเทคนิคที่ผู้โจมตีใช้ในการย้ายแปลงตรรกะแบบเครือข่ายหลังจากเข้าถึงครั้งแรก ซึ่งรวมถึงการโจมตีจุดอ่อนในส่วนต่างๆ ของเครือข่ายและการใช้ข้อมูลประจำที่ถูกโจมตีเพื่อเข้าถึงทรัพยากร ผู้โจมตีอาจใช้เครื่องมือ เช่น PsExec หรือ PowerShell สำหรับการเผยแพร่ด้านข้าง

### การใช้ประโยชน์จากโปรโตคอลเครือข่าย
เกี่ยวข้องกับการระบุจุดอ่อนในโปรโตคอลเครือข่าย เช่น SMB, RDP หรือ DNS และการใช้ประโยชน์จากพวกเขาเพื่อดำเนินการที่ไม่ได้รับอนุญาต ความรู้เกี่ยวกับกลไกโปรโตคอลและการสร้างความผิดปกติสามารถนำไปสู่การใช้ประโยชน์สำเร็จได้

### เทคนิคการโจมตีประเภท Man-in-the-Middle (MitM)
การโจมตี MitM เกี่ยวข้องกับการปิดกั้นและแก้ไขการสื่อสารระหว่างสองฝ่ายโดยไม่ให้ถูกตรวจพบ เทคนิคอาจรวมถึงการปลอมแปลง ARP หรือการวางยาห่า DNS ใช้เครื่องมือเช่น Ettercap และ MITMf ในการจับและแก้ไขทราฟฟิก

### การวิเคราะห์และการจัดการทราฟฟิกเครือข่าย
การวิเคราะห์ทราฟฟิกเกี่ยวข้องกับการสอดส่องแพ็กเก็ตข้อมูลเครือข่ายเพื่อระบุรูปแบบหรือข้อบกพร่อง ผู้โจมตีอาจบิดเบือน ตัด หรือฉีดแพ็กเก็ตเพื่อลดหรือแยกการสื่อสาร เครื่องมืออย่าง Tcpdump และ Scapy มักใช้กันอย่างแพร่หลาย

## แอปพลิเคชันเว็บ

### การโจมตีด้วยการฉีดขั้นสูง (SQLi, NoSQLi)
SQL Injection เกี่ยวข้องกับการเพิ่มเติมโค้ด SQL ที่อันตรายลงในฟิลด์อินพุตของผู้ใช้เพื่อบิดเบือนหรือดึงข้อมูลจากฐานข้อมูล หลักการนี้ขยายไปยังฐานข้อมูล NoSQL ด้วย NoSQLi SQLmap เป็นเครื่องมือที่ใช้กันทั่วไปในการทำอัตโนมัติการฉีด SQL และการยึดครองฐานข้อมูล:

```bash
sqlmap -u "http://target.com/search?q=test" --dbs
```

คำสั่งนี้พยายามดึงชื่อฐานข้อมูลจาก URL เป้าหมาย

### การแอบแฝงของ XSS และการผูกมัด CSRF
XSS อนุญาตให้ผู้โจมตีแทรกสคริปต์ลงในหน้าเว็บที่ผู้ใช้อื่นดูเพื่อเจาะกล้องหรือขโมยข้อมูล CSRF หลอกผู้ใช้ให้ดำเนินการที่ไม่ต้องการบนเว็บไซต์อื่นที่ได้รับการยืนยัน

### การอ้างอิงวัตถุโดยตรงที่ไม่ปลอดภัยและ BOLAs
ช่องโหว่ IDOR อนุญาตให้ผู้โจมตีเข้าถึงข้อมูลโดยการปรับการอ้างอิงไปยังวัตถุ (เช่น รหัสผู้ใช้) การอนุญาตให้ใช้ข้อผิดพลาดช่องทางระดับวัตถุ (BOLAs) เกี่ยวข้องกับการบังคับใช้อย่างไม่เพียงพอของการควบคุมการเข้าถึง

### ยุทธวิธีเว็บเชลล์และการยืนยันตน
ติดตั้งเว็บเชลล์บนเซิร์ฟเวอร์ที่ถูกโจมตีเพื่อการเข้าถึงที่ต่อเนื่องและการดำเนินการคำสั่งจากระยะไกล เว็บเชลล์เป็นสคริปต์ง่าย ๆ ที่ให้ส่วนเชื่อมต่อผ่าน HTTP

```php
<?php system($_GET['cmd']); ?>
```

ตัวอย่างของเว็บเชลล์ PHP อย่างง่าย

### การหาทางหลีกเลี่ยงไฟร์วอลล์แอปพลิเคชันเว็บ
การหลีกเลี่ยง WAF รวมถึงการออกแบบเพย์โหลดให้หลีกเลี่ยงการตรวจพบรูปแบบ-ตามลักษณะ เทคนิคที่ใช้ได้แก่ การเข้ารหัส การคลุมเคลือ หรือการแยกส่วนแกนการโจมตี

## แอปพลิเคชันมือถือ

### วิศวกรรมย้อนกลับแอปพลิเคชันมือถือ
วิศวกรรมย้อนกลับเกี่ยวข้องกับการถอดรหัสและวิเคราะห์โค้ดของแอปมือถือเพื่อเข้าใจตรรกะของมันและระบุช่องโหว่ เครื่องมือเช่น APKTool หรือ Hopper เป็นสิ่งจำเป็นในการถอดส่วนประกอบหรือถอดแยกโค้ด

### การใช้ประโยชน์จากช่องโหว่แพลตฟอร์มมือถือ
แต่ละแพลตฟอร์มมือถือ (เช่น Android, iOS) มีช่องโหว่เฉพาะ การโจมตีอาจรวมถึงการใช้ประโยชน์จากสิทธิ์ของระบบหรือห้องสมุดเก่าที่ไม่ได้อัพเดต

### การสกัดกั้นและการจัดการการสื่อสารบนมือถือ
ผู้โจมตีใช้เครื่องมือเช่น Burp Suite หรือ Frida เพื่อสกัดกั้นและจัดการข้อมูลที่ส่งผ่านแอปพลิเคชันมือถือ โดยเฉพาะเมื่อการใช้งาน SSL/TLS อ่อนแอหรือไม่ถูกตั้งค่าอย่างถูกต้อง

### ข้อมูลส่วนการบุกรุกการรักษาความปลอดภัยแอปพลิเคชันมือถือ
มุ่งเป้าหมายไปที่การจัดเก็บข้อมูลที่ไม่ปลอดภัยหรือวิธีการส่งข้อมูลที่ใช้โดยแอป ซึ่งจัดเก็บหรือสื่อสารถข้อมูลสำคัญโดยไม่มีการเข้ารหัสที่เพียงพอ

## ห่วงโซ่อุปทาน

### การโจมตีที่เป้าหมายการพัฒนาและเส้นทางการส่งมอบ
การเจาะรหัสในเส้นทางการพัฒนาพร้อมสามารถนำไปสู่การแจกจ่ายโค้ดที่เป็นอันตราย ทีมโจมตีมุ่งเป้าหมายไปที่ที่เก็บรหัสหรือการตั้งค่าทำงาน CI/CD เพื่อแอบวางประตูหลัง

### การทดสอบความปลอดภัยของระบบสมองกลฝังตัว
เน้นการระบุช่องโหว่ในเฟิร์มแวร์หรือซอฟต์แวร์ของอุปกรณ์สมองกลฝังตัว การทดสอบมักครอบคลุมอินเทอร์เฟซฮาร์ดแวร์เช่น JTAG หรือ UART 

### เทคนิคการแทรกซึมห่วงโซ่อุปทาน
วางชิ้นส่วนที่เป็นอันตรายภายในห่วงโซ่อุปทานของสินค้าเพื่อทำลายความสมบูรณ์ก่อนที่สินค้าจะไปถึงผู้ใช้ปลายทาง กรณีที่โดดเด่นรวมถึงการฝังฮาร์ดแวร์หรือไฟล์อัพเดตที่ถูกดัดแปลง 

### การวางประตูหลังในซอฟต์แวร์ของบุคคลที่สาม
พรางวางรหัสที่ไม่ได้รับอนุญาตลงในผลิตภัณฑ์ซอฟต์แวร์ที่ซัพพลายโดยผู้ขายบุคคลที่สามระหว่างช่วงพัฒนา ตั้งใจหรือผ่านการแสวงหาผลประโยชน์ 

## CI/CD

### การบุกรุกสภาพแวดล้อม CI/CD
การเข้าถึงสภาพแวดล้อม CI/CD ที่ไม่ได้รับอนุญาตให้อำนาจในการตัดต่อกระบวนการสร้างซอฟต์แวร์ อาจแทรกรหัสที่เป็นอันตรายก่อนการประยุกต์

### การแทรกซึมในเส้นทางการสร้าง
ผู้โจมตีดำเนินการยึดหรือเปลี่ยนแปลงไฟล์ตกแต่ง ซึ่งนำไปสู่การแจกจ่ายซอฟต์แวร์ที่ถูกโจมตี ช่องโหว่ในสคริปต์การสร้างหรือการพึ่งพามักถูกใช้ในการโจมตี

### การปรับแก้ไฟล์สำเร็จและใส่ร้ายความสมบูรณ์
การเปลี่ยนแปลงไฟล์สำเร็จที่ไม่มีการตรวจจับสามารถทำลายฟังก์ชันที่ตั้งใจไว้ของซอฟต์แวร์ได้ การยืนยันความสมบูรณ์ของไฟล์สำเร็จเป็นสิ่งสำคัญ โดยอาศัยการตรวจสอบแฮชหรือการเซ็นลายเซ็นทางคริปโต

### การใช้ประโยชน์จากช่องโหว่เครื่องมือ CI/CD
ช่องโหว่ที่รู้จักในเครื่องมือ CI/CD เช่น Jenkins หรือ GitLab สามารถถูกใช้ประโยชน์เพื่อเข้าถึงที่ไม่ได้รับอนุญาตและแก้ไขรหัส

## คลาวด์

### เทคนิคการละเมิดบริการคลาวด์
การทำลายหรือใช้ช่องโหว่ในบริการคลาวด์เพื่อลักลอบเข้าถึงหรือควบคุมทรัพยากรที่จัดการโดยคลาวด์

### การโจมตีคลังจัดเก็บคลาวด์ที่ไม่ได้ตั้งค่าอย่างถูกต้อง
มุ่งเป้าหมายที่บริการจัดเก็บที่เปิดอยู่เช่น AWS S3 ที่ไม่เพียงพอ แต่เกี่ยวข้องกับการควบคุมความปลอดภัย เช่น การอนุญาตให้อ่านหรือเขียนสาธารณะ

### การหลบหนีจากการจัดการข้อมูลประจำตัวและการเข้าถึงคลาวด์
ได้รับการเข้าถึงที่ไม่ได้รับอนุญาตไปยังทรัพย์สินของคลาวด์โดยใช้ประโยชน์จากนโยบาย IAM ที่อ่อนแอหรือบทบาทที่เปิดเผยเกินไป

### การโจมตีเวกเตอร์ในสถาปัตยกรรมเซิร์ฟเวอร์เลส
การใช้ประโยชน์จากช่องโหว่ในกรอบการทำงานเซิร์ฟเวอร์เลสโดยใช้เหตุการณ์ที่เรียกใช้หรือการนำเพย์โหลดอันตรายเข้าไปในบริบทการทำงานของฟังก์ชัน

## คอนเทนเนอร์

### เทคนิคการหลบหนีจากคอนเทนเนอร์
ระบุและใช้ประโยชน์จากช่องโหว่ที่อนุญาตให้โค้ดอันตรายหลุดออกจากสภาพแวดล้อมคอนเทนเนอร์เข้าสู่ระบบโฮสต์

### การโจมตีกลุ่ม Kubernetes
การใช้ประโยชน์จากการตั้งค่าผิดในสภาพแวดล้อม Kubernetes เพื่อให้ควบคุมโดยไม่ได้รับอนุญาตหรือการรบกวนบริการ

### การใช้ประโยชน์จากช่องโหว่ของการจัดการคอนเทนเนอร์
มุ่งเป้าหมายไปที่ช่องโหว่ในเครื่องมือการจัดการเช่น Docker Swarm หรือ Kubernetes เพื่อเข้าถึงสภาพแวดล้อมที่จำกัด

### การวางยาพิษภาพคอนเทนเนอร์
ปรับเปลี่ยนภาพคอนเทนเนอร์โดยฉีดโค้ดอันตราย ซึ่งส่งผลกระทบต่อระบบที่ภาพเหล่านั้นถูกติดตั้ง

## API

### การระบุและสำรวจช่องทาง API
ระบุช่องทาง API และสร้างแผนโครงสร้างของพวกเขาเพื่อกำหนดทรัพยากรที่มีและช่องโหว่ที่เป็นไปได้

### การหลีกเลี่ยงการตรวจสอบและการอนุญาตของ API
ใช้ประโยชน์จากช่องโหว่ที่อนุญาตให้หลีกเลี่ยงเลเยอร์การตรวจสอบหรือข้อกำหนดการอนุญาตที่ไม่ถูกต้อง

### การละเมิดจำกัดการร้องขอและโควต้า
หลีกเลี่ยงข้อจำกัดการร้องขอเพื่อทำให้ API ล้นหรือดึงข้อมูลมากกว่าที่ตั้งใจไว้โดยการปรับหัวหรือข้อมูลที่ส่งไป

### การใช้ประโยชน์จากช่องโหว่ REST และ SOAP API
มุ่งเป้าหมายไปที่ API เฉพาะผ่านการฉีด, การโจมตีการคลายข้อมูล หรือการใช้ประโยชน์จากข้อผิดพลาดที่เกิดจากข้อความผิดพลาดที่verbose

## ฮาร์ดแวร์

### การใช้ประโยชน์จากอินเทอร์เฟซฮาร์ดแวร์
การจัดการหรือเข้าถึงอินเทอร์เฟซฮาร์ดแวร์เพื่อดึงข้อมูลที่สำคัญหรือแก้ไขการทำงานของอุปกรณ์

### การโจมตีช่องทางด้านข้างบนฮาร์ดแวร์
การใช้ประโยชน์จากการปล่อยคลื่นแม่เหล็กไฟฟ้า, การวิเคราะห์เวลาการบริโภคพลังงาน หรือข้อมูลการจับเวลาเพื่อติดตามการทำงานของอุปกรณ์หรือดึงข้อมูลสำคัญ

### การละเมิดอุปกรณ์ฝังตัว
ได้เข้าถึงอุปกรณ์ฝังตัวที่ไม่ได้รับอนุญาตผ่านช่องโหว่ในเฟิร์มแวร์หรืออินเทอร์เฟซการสื่อสารภายนอก

### วิศวกรรมย้อนกลับเฟิร์มแวร์ฮาร์ดแวร์
การวิเคราะห์เฟิร์มแวร์ของอุปกรณ์ฮาร์ดแวร์เพื่อตระหนักถึงช่องโหว่หรือเข้าใจการทำงานของอุปกรณ์ผ่านวิศวกรรมย้อนกลับ

## ไร้สาย

### การโจมตีเครือข่ายไร้สาย (Wi-Fi, Bluetooth)
การใช้ประโยชน์จากโปรโตคอล Wi-Fi ที่อ่อนแอ (WEP/WPA2) หรือการตั้งค่า Bluetooth ที่ไม่ได้ตั้งค่าอย่างถูกต้องเพื่อสกัดกั้นและจัดการการสื่อสารไร้สาย

### การวางแถวของจุดสำรวจที่หลอกลวง
การวางจุดสำรวจที่ไม่ได้รับอนุญาตเพื่อสกัดกั้นการสื่อสารไร้สาย เทคนิคที่ใช้กันบ่อยๆ ในการฟิชชิ่งหรือการขโมยข้อมูลประจำตัว

### การแอบฟังการสื่อสารไร้สาย
การฟังแบบไม่ได้เข้ารหัสบนเครือข่ายไร้สายเพื่อจับข้อมูลสำคัญเช่นรหัสผ่าน

### การใช้ประโยชน์จากโปรโตคอลไร้สาย
ใช้ประโยชน์จากจุดอ่อนในโปรโตคอลเช่น Zigbee หรือ Bluetooth Low Energy (BLE) เพื่อควบคุมหรือสอดส่องการสื่อสารของอุปกรณ์

## การโจมตีความปลอดภัยทางกายภาพ

### วิศวกรรมสังคมและการแทรกซึมทางกายภาพ
ใช้กลลวงเพื่อผ่านมาตรการความปลอดภัยทางกายภาพ ซึ่งมีการปลอมแปลงหรือบิดเบือนข้อมูลประจำตัวเพื่อได้เข้าถึงที่ไม่ได้รับอนุญาตในอาคาร

### การเลี่ยงระบบควบคุมการเข้าถึงทางกายภาพ
การหลีกเลี่ยงข้อกีดขวางเช่นล็อคหรือระบบชีวมิติผ่านเทคนิคเช่นการถอดล็อคหรือการโจมตีแรงกระตุ้น

### การขโมยข้อมูลชั้นความปลอดภัยทางกายภาพ
การดึงข้อมูลผ่านวิธีการทางกายภาพ เช่น การคัดลอกข้อมูลจากฮาร์ดไดรฟ์ที่เปิดเผยหรือการสกัดกั้นสัญญาณแม่เหล็กไฟฟ้า

### วิธีการประเมินความปลอดภัยทางกายภาพ
เทคนิคตามการจำลองเพื่อประเมินช่องโหว่ในตั้งค่าความปลอดภัยทางกายภาพ ซึ่งมีการทดสอบการแทรกซึมของที่ตั้ง

## คริปโตกราฟฟิก

### การโจมตีโปรโตคอลคริปโตกราฟฟิก
ใช้ประโยชน์จากช่องโหว่ในโปรโตคอลคริปโตกราฟฟิก ให้สามารถถอดรหัสข้อมูล, การเลี่ยงการรับรองตัวตน, หรือการแอบอ้างตัว

### เทคนิควิศวกรรมคริปโดยิสิต
เทคนิคที่ใช้การวิเคราะห์ทางคณิตศาสตร์และเชิงระบบเพื่อทำลายความปลอดภัยทางคริปโตกราฟฟิก ซึ่งรวมถึงการทำลายการเข้ารหัส

### การละเมิดการจัดการคีย์
การโจมตีการปฏิบัติการจัดการคีย์ที่ไม่เหมาะสม ซึ่งนำไปสู่การเปิดเผยคีย์ที่ไม่ถูกต้องและการถอดรหัสข้อมูล

### การโจมตีการเข้ารหัสสมมาตรและอสมมาตร
ค้นหาจุดอ่อนหรือใช้ประโยชน์จากข้อบกพร่องในทางปฏิบัติเพื่อถอดรหัสข้อมูลโดยไม่ต้องครอบครองคีย์ เขตการโจมตีที่รู้จักรวมถึงการโจมตี AES หรือ RSA

## การพัฒนาช่องโหว่

### การระบุโอกาสในการใช้ประโยชน์จากช่องโหว่
วิจัยและค้นพบช่องโหว่ในซอฟต์แวร์หรือฮาร์ดแวร์ที่สามารถใช้ประโยชน์ได้ มุ่งเน้นไปที่จุดเข้าในอินพุตที่ไม่ได้กรองหรือการดำเนินการที่ไม่ได้ตรวจสอบ

### เทคนิคการใช้ประโยชน์จากหน่วยความจำขั้นสูง
การใช้จุดอ่อนในหน่วยความจำเพื่อสั่งการโค้ดตามต้องการ เช่น การบุกรุกบัฟเฟอร์และการใช้การคืนค่าที่ไม่ถูกต้อง

### การสร้างและส่งมอบ Shellcode
ออกแบบและฉีด shellcode เป็นเพย์โหลดเพื่อทำการสั่งการที่เป็นอันตรายในระบบเป้าหมาย

### การใช้เฟรมเวิร์กการโจมตี
ใช้เครื่องมือเช่น Metasploit เพื่อช่วยพัฒนาและจัดการช่องทางการโจมตีผ่านโมดูลที่สามารถปรับแต่งได้สำหรับเป้าหมายที่หลากหลาย

## Red Teaming

### การวางแผนและดำเนินการ Red Team Engagements
Red teaming เกี่ยวข้องกับการจำลองภัยคุกคามขั้นสูงเพื่อประเมินการป้องกันขององค์กร การวางแผนเน้นไปที่การบรรลุวัตถุประสงค์ยุทธวิธี เทคนิค และขั้นตอน (TTPs) เฉพาะ

### การตั้งค่าโครงสร้างพื้นฐานและเครื่องมือ Red Team
สร้างโครงสร้างพื้นฐานที่ปลอดภัยและยืดหยุ่นสำหรับการดำเนินการ red team เน้นที่ระบบคำสั่งและควบคุม (C2) และเครื่องมือโจมตี

### แคมเปญการโจมตีหลายขั้นตอน
แคมเปญที่วางแผนไว้อย่างประสาน เช่นเดียวกับผู้โจมตีในโลกความจริง ที่เกี่ยวข้องกับหลายขั้นตอนเช่น การสำรวจ การโจมตี และการเคลื่อนไหวด้านข้าง

### การจำลองและเอมูลแอดีโอซารี
การจำลองวิธีที่แท้จริงที่ใช้โดยผู้คุกคามขั้นสูงเพื่อประเมินมาตรการป้องกันขององค์กรและระบุพื้นที่ที่ต้องปรับปรุง

## การหลีกเลี่ยง

### การหลีกเลี่ยงการตรวจจับเครือข่ายและปลายทาง
ใช้เทคนิคที่จะหลีกเลี่ยงการตรวจจับโดย IDS/IPS และโซลูชั่นความปลอดภัยปลายทาง ผ่านการคลุมเคลือของมัลแวร์หรือการใช้ประโยชน์ช่องโหว่

### การต่อต้านการพิสูจน์และการจัดการหลักฐาน
เทคนิคที่ออกแบบมาเพื่อตอบโต้การวิเคราะห์ทางนิติวิทยาศาสตร์หรือทำลายลบหลักฐาน ซึ่งมีความสำคัญในการรักษาการปฏิบัติการที่เป็นความลับ

### เทคนิคการคลุมเคลือและการเข้ารหัส
การปกปิดเพย์โหลดหรือการสื่อสารผ่านการคลุมเคลือหรือการเข้ารหัสเพื่อหลีกเลี่ยงมาตรการความปลอดภัยและการวิเคราะห์

### การหลีกเลี่ยง Honeypot และ Sandbox
การตรวจจับและหลีกเลี่ยงการโต้ตอบกับระบบหลอกที่ออกแบบมาเพื่อจับภาพกิจกรรมที่เป็นอันตราย เช่น honeypots หรือสภาพแวดล้อมsandbox

## การพัฒนามัลแวร์

### การออกแบบและส่งมอบเพย์โหลดที่กำหนดเอง
การสร้างเพย์โหลดที่ปรับแต่งเอง คือต้องการเจาะเฉพาะสภาพแวดล้อมหรือหลีกเลี่ยงการป้องกันที่มีอยู่ มุ่งเน้นความระแวดระวังและการใช้งาน

### เทคนิคการหลีกเลี่ยงและความพออดทนของมัลแวร์
การสร้างมัลแวร์ให้คงไว้ในระบบหลังจากการลุกลามและหลีกเลี่ยงการตรวจจับ ผ่านการคลุมเคลือหรือการจัดการการป้องกันโฮสต์

### การพัฒนาและการจัดการโครงสร้างคำสั่งและควบคุม
การสร้างและจัดการโครงสร้างคำสั่งและควบคุมเพื่อรักษาความเป็นอิสระจากระบบที่ถูกโจมตี ซึ่งสำคัญสำหรับตัวละครคุกคามที่ยืนหยัด

### วิธีการข้ามหลอดซอฟต์แวร์ป้องกันมัลแวร์
ความพยายามที่จะหลีกเลี่ยงการตรวจจับมัลแวร์อัตโนมัติผ่านเทคนิคการหลีกเลี่ยงเช่น การเปลี่ยนแปลงลายเซ็นหรือการปรับเปลี่ยนพฤติกรรม

## ซอฟต์แวร์

### เทคนิคการวางทางลับซอฟต์แวร์
การวางทางลับในโค้ดซอฟต์แวร์เพื่อให้เข้าถึงที่ไม่ได้รับอนุญาต มักเกิดขึ้นในการจัดการโค้ดแบบโอเพนซอร์สหรือโค้ดที่มีลิขสิทธิ์

### การจัดการตรรกะของแอปพลิเคชัน
ใช้ประโยชน์จากข้อผิดพลาดในตรรกะธุรกิจหรือการไหลของแอปพลิเคชันเพื่อดำเนินการที่ไม่ได้ตั้งใจหรือดึงข้อมูลอย่างไม่ถูกต้อง

### ยุทธวิธีการย้อนกลับขั้นสูง
การวิเคราะห์ซอฟต์แวร์ที่ลึกลงไปโดยใช้เครื่องมือเช่น IDA Pro หรือ Ghidra เพื่อเปิดเผยตรรกะภายในหรือช่องโหว่ในระบบปิด

### การฉีดและการแทรกแซงโค้ด
การฉีดโค้ดที่เป็นอันตรายลงในกระบวนการที่กำลังทำงานหรือการสกัดกั้นการไหลของงานของแอปผ่านเทคนิคการแทรกแซง มักใช้ในการปรับระดับสิทธิ
