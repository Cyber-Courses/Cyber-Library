# Peta Jalan Keamanan Siber Ofensif

## Jaringan

### Pengintaian dan Pemetaan Jaringan
Pengintaian jaringan melibatkan pengumpulan informasi tentang jaringan target untuk menciptakan peta struktur dan perangkatnya. Teknik ini mencakup pengumpulan informasi secara pasif, pemindaian, dan enumerasi aktif dari komponen jaringan. Alat seperti Nmap dan Wireshark digunakan untuk mengidentifikasi port terbuka, layanan, dan potensi kerentanan. Gunakan perintah seperti:

```bash
nmap -sS -O -v target-ip
```

Perintah ini melakukan scan TCP SYN dan mendeteksi sistem operasi target.

### Strategi Pergerakan Lateral
Pergerakan lateral mengacu pada teknik yang digunakan penyerang untuk berpindah melalui jaringan setelah mendapatkan akses awal. Ini termasuk eksploitasi kerentanan di segmen jaringan yang berdekatan dan menggunakan kredensial yang telah dikompromikan untuk mengakses sumber daya. Penyerang mungkin menggunakan alat seperti PsExec atau PowerShell untuk penyebaran lateral.

### Eksploitasi Protokol Jaringan
Ini melibatkan identifikasi kelemahan dalam protokol jaringan seperti SMB, RDP, atau DNS dan memanfaatkannya untuk melakukan tindakan yang tidak sah. Pengetahuan tentang mekanisme protokol dan pembuatan anomali dapat mengarah pada eksploitasi yang berhasil.

### Teknik Serangan Man-in-the-Middle (MitM)
Serangan MitM melibatkan penyadapan dan pengubahan komunikasi antara dua pihak tanpa terdeteksi. Teknik ini dapat mencakup spoofing ARP atau poisoning DNS. Penyerang memanfaatkan alat seperti Ettercap dan MITMf untuk menangkap dan memanipulasi lalu lintas.

### Analisis dan Manipulasi Lalu Lintas Jaringan
Analisis lalu lintas melibatkan pemantauan paket data jaringan untuk mengidentifikasi pola atau anomali. Penyerang mungkin mengalihkan, menjatuhkan, atau menyuntikkan paket untuk mengganggu atau membongkar komunikasi. Alat seperti Tcpdump dan Scapy umumnya digunakan untuk tujuan ini.

## Aplikasi Web

### Serangan Injeksi Lanjutan (SQLi, NoSQLi)
Injeksi SQL melibatkan penyisipan kode SQL berbahaya ke dalam bidang input pengguna untuk memanipulasi atau mengekstraksi data dari database. Prinsip ini diperluas ke database NoSQL dengan NoSQLi. SQLmap adalah alat umum untuk otomatisasi injeksi SQL dan pengambilalihan database:

```bash
sqlmap -u "http://target.com/search?q=test" --dbs
```

Perintah ini mencoba mengambil nama database dari URL target.

### Eksploitasi XSS dan CSRF
XSS memungkinkan penyerang untuk menyuntikkan skrip ke halaman web yang dilihat oleh pengguna lain, memungkinkan pembajakan sesi atau pencurian data. CSRF menipu pengguna untuk mengeksekusi tindakan yang tidak diinginkan di situs lain tempat mereka diautentikasi.

### IDOR dan BOLA
Kerentanan IDOR memungkinkan penyerang mengakses data dengan memanipulasi referensi ke objek (misalnya ID pengguna). Broken Object Level Authorization (BOLA) melibatkan penegakan yang tidak tepat dari kontrol akses atas objek individual.

### Taktik Web Shell dan Persistensi
Memasang web shell di server yang telah dikompromikan untuk akses yang persisten dan eksekusi perintah jarak jauh. Web shell adalah skrip sederhana yang menyediakan antarmuka melalui HTTP.

```php
<?php system($_GET['cmd']); ?>
```

Contoh web shell PHP yang sederhana.

### Penghindaran WAF Aplikasi Web
Menghindari WAF melibatkan pembuatan payload yang melewati deteksi berbasis tanda tangan. Teknik ini mencakup pengkodean, pengaburan, atau fragmentasi vektor serangan.

## Aplikasi Seluler

### Rekayasa Balik Aplikasi Seluler
Rekayasa balik melibatkan dekripsi dan analisis kode aplikasi seluler untuk memahami logikanya dan mengidentifikasi kerentanan. Alat seperti APKTool atau Hopper sangat penting untuk disassembly atau dekompilasi.

### Eksploitasi Kerentanan Platform Seluler
Setiap platform seluler (misalnya, Android, iOS) memiliki kerentanan khusus. Eksploitasi mungkin melibatkan pemanfaatan hak istimewa sistem atau pustaka yang ketinggalan zaman.

### Penyadapan dan Manipulasi Komunikasi Seluler
Penyerang menggunakan alat seperti Burp Suite atau Frida untuk mencegat dan memanipulasi data yang dikirim oleh aplikasi seluler, terutama ketika implementasi SSL/TLS lemah atau salah konfigurasikan.

### Pelanggaran Keamanan Data Aplikasi Seluler
Menargetkan penyimpanan data yang tidak aman atau metode transmisi data yang digunakan oleh aplikasi seluler, di mana informasi sensitif disimpan atau disampaikan tanpa enkripsi yang memadai.

## Rantai Pasokan

### Menargetkan Pengembangan dan Jalur Pengantaran
Mengompromikan jalur pengembangan dapat menyebabkan distribusi kode berbahaya yang luas. Penyerang menargetkan repositori kode sumber atau lingkungan CI/CD untuk menyisipkan backdoors.

### Pengujian Keamanan Sistem Tertanam
Fokus pada mengidentifikasi kerentanan dalam firmware atau perangkat lunak perangkat tertanam. Pengujian sering mencakup antarmuka perangkat keras seperti JTAG atau UART.

### Teknik Kompromi Rantai Pasokan
Menyisipkan komponen berbahaya dalam rantai pasokan produk mengganggu integritas sebelum sampai ke pengguna akhir. Contoh terkenal termasuk implant hardware atau file pembaruan yang disabotase.

### Backdooring Perangkat Lunak Pihak Ketiga
Memperkenalkan kode tersembunyi dan tidak sah ke dalam produk perangkat lunak yang disuplai oleh vendor pihak ketiga selama pengembangan, baik dengan sengaja atau melalui eksploitasi.

## CI/CD

### Mengompromikan Lingkungan CI/CD
Akses tidak sah ke lingkungan CI/CD memberikan kekuatan untuk mengubah proses build perangkat lunak, berpotensi menyisipkan kode berbahaya sebelum deployment.

### Penyusupan Jalur Build
Penyerang mencegat atau memodifikasi artefak build, yang mengarah pada distribusi perangkat lunak yang dikompromikan. Kerentanan dalam skrip build atau ketergantungan sering kali dieksploitasi.

### Pemalsuan Artefak dan Kompromi Integritas
Modifikasi artefak build yang tidak terdeteksi dapat merusak fungsionalitas perangkat lunak yang dimaksudkan. Memastikan integritas artefak sangat penting, mengandalkan checksum atau tanda tangan kriptografi.

### Eksploitasi Kerentanan Alat CI/CD
Kerentanan yang diketahui dalam alat CI/CD, seperti Jenkins atau GitLab, dapat dieksploitasi untuk akses tidak sah dan pengubahan kode.

## Cloud

### Teknik Eksploitasi Layanan Cloud
Memanfaatkan konfigurasi yang salah atau kerentanan dalam layanan cloud untuk mendapatkan akses atau kontrol yang tidak sah atas sumber daya yang dikelola cloud.

### Serangan Penyimpanan Cloud yang Salah Konfigurasi
Menargetkan layanan penyimpanan yang terekspos seperti bucket AWS S3 karena kontrol keamanan yang tidak memadai seperti akses tulis atau baca publik.

### Bypass Identitas dan Manajemen Akses Cloud
Mendapatkan akses tidak sah ke aset cloud dengan mengeksploitasi kebijakan IAM yang lemah atau peran yang terlalu permisif.

### Vektor Serangan Arsitektur Tanpa Server
Mengeksploitasi kerentanan dalam kerangka kerja serverless dengan menyalahgunakan pemicu acara atau menyuntikkan payload berbahaya ke dalam konteks eksekusi fungsi.

## Kontainer

### Teknik Pelarian Kontainer
Mengidentifikasi dan mengeksploitasi kerentanan yang memungkinkan kode berbahaya untuk keluar dari lingkungan kontainer ke sistem host.

### Menyerang Klaster Kubernetes
Mengeksploitasi konfigurasi yang salah dalam lingkungan Kubernetes untuk mendapatkan kontrol yang tidak sah atau mengganggu layanan.

### Eksploitasi Kerentanan Orkestrasi Kontainer
Menargetkan kerentanan dalam alat orkestrasi seperti Docker Swarm atau Kubernetes untuk mengakses lingkungan terbatas.

### Meracuni Citra Kontainer
Memanipulasi citra kontainer dengan menyuntikkan kode berbahaya, mempengaruhi sistem di mana citra tersebut diterapkan.

## API

### Enumerasi dan Pengintaian Endpoint API
Mengidentifikasi endpoint API dan memetakan struktur mereka untuk menentukan sumber daya yang tersedia dan potensi kerentanan.

### Bypass Otentikasi dan Otorisasi API
Mengeksploitasi kerentanan yang memungkinkan bypass lapisan otentikasi atau pemeriksaan otorisasi yang tidak tepat.

### Penyalahgunaan Batas Tarif dan Kuota
Melewati batas tarif untuk membanjiri API atau mengekstrak lebih banyak data dari yang dimaksudkan dengan memanipulasi header atau payload permintaan.

### Eksploitasi Kerentanan API REST dan SOAP
Menargetkan API tertentu melalui injeksi, serangan deserialisasi, atau memanfaatkan paparan akibat pesan kesalahan yang terlalu rinci.

## Perangkat Keras

### Eksploitasi Antarmuka Perangkat Keras
Memanipulasi atau mengakses antarmuka perangkat keras untuk mengambil data sensitif atau memodifikasi operasi perangkat.

### Serangan Saluran Samping pada Perangkat Keras
Memanfaatkan emisi elektromagnetik, analisis konsumsi daya, atau informasi waktu untuk menyimpulkan operasi perangkat atau mengambil informasi sensitif.

### Menembus Perangkat Tertanam
Mendapatkan akses tidak sah ke perangkat tertanam melalui kerentanan dalam firmware atau antarmuka komunikasi eksternal.

### Rekayasa Balik Firmware Perangkat Keras
Menganalisis firmware perangkat keras untuk mengidentifikasi kerentanan atau memahami operasi perangkat melalui rekayasa balik.

## Nirkabel

### Serangan Jaringan Nirkabel (Wi-Fi, Bluetooth)
Mengeksploitasi protokol Wi-Fi yang lemah (WEP/WPA2) atau konfigurasi yang salah pada Bluetooth untuk mencegat dan memanipulasi komunikasi nirkabel.

### Penyebaran Titik Akses Palsu
Menyiapkan titik akses tidak sah untuk mencegat komunikasi nirkabel, sebuah teknik yang sering digunakan dalam phishing atau pencurian kredensial.

### Penyadapan pada Komunikasi Nirkabel
Mendengarkan komunikasi yang tidak terenkripsi melalui jaringan nirkabel untuk menangkap informasi sensitif seperti kata sandi.

### Eksploitasi Protokol Nirkabel
Mengeksploitasi kelemahan dalam protokol seperti Zigbee atau Bluetooth Low Energy (BLE) untuk mendapatkan kontrol atau memata-matai komunikasi perangkat.

## Serangan Keamanan Fisik

### Rekayasa Sosial dan Penetrasi Fisik
Menggunakan tipu daya untuk melewati langkah-langkah keamanan fisik, melibatkan penyamaran atau pemalsuan kredensial untuk mendapatkan akses tidak sah ke fasilitas.

### Menghindari Kontrol Akses Fisik
Melewati penghalang seperti kunci atau sistem biometrik melalui teknik seperti pengambil kunci atau serangan relay.

### Ekstraksi Informasi Lapisan Fisik
Mengambil data melalui cara fisik, seperti menyalin data dari hard drive yang terekspos atau mencegat sinyal elektromagnetik.

### Metode Penilaian Keamanan Fisik
Teknik berbasis simulasi untuk menilai kerentanan dalam pengaturan keamanan fisik, melibatkan pengujian penetrasi di tempat.

## Kriptografi

### Serangan Protokol Kriptografi
Mengeksploitasi kerentanan dalam protokol kriptografi, yang mengarah pada dekripsi data, bypass otentikasi, atau penyamaran.

### Teknik Kriptanalisis
Teknik melibatkan analisis matematis dan sistemik untuk mengkompromikan keamanan kriptografi, termasuk pemecahan sandi.

### Penyalahgunaan Manajemen Kunci
Menyerang praktik manajemen kunci yang tidak tepat yang mengarah pada pengungkapan kunci yang tidak sah dan dekripsi data.

### Memecahkan Enkripsi Simetrik dan Asimetrik
Menemukan kelemahan atau mengeksploitasi kelemahan implementasi untuk mendekripsi data tanpa kepemilikan kunci. Contohnya termasuk serangan terhadap AES atau RSA.

## Pengembangan Eksploitasi

### Mengidentifikasi Peluang Eksploitasi Kerentanan
Meneliti dan menemukan kerentanan dalam perangkat lunak atau perangkat keras yang dapat dieksploitasi. Fokus pada titik masuk dalam input yang tidak disanitasi atau operasi yang tidak diperiksa.

### Teknik Korupsi Memori Lanjutan
Memanfaatkan kelemahan dalam operasi memori untuk mencapai eksekusi kode sewenang-wenang, termasuk buffer overflow dan eksploitasi use-after-free.

### Pembuatan dan Pengiriman Shellcode
Merancang dan menyuntikkan shellcode sebagai payload untuk mengeksekusi instruksi berbahaya pada sistem target.

### Pemanfaatan Kerangka Eksploitasi
Menggunakan alat seperti Metasploit untuk membantu pengembangan eksploitasi dan mengelola vektor serangan melalui modul yang dapat dikonfigurasi untuk berbagai target.

## Red Teaming

### Perencanaan dan Eksekusi Keterlibatan Tim Merah
Red teaming melibatkan simulasi ancaman persisten tingkat lanjut (APT) untuk mengevaluasi pertahanan organisasi. Perencanaan difokuskan pada pencapaian taktik, teknik, dan prosedur (TTP) tertentu.

### Pengaturan Infrastruktur dan Alat Tim Merah
Membangun infrastruktur yang aman dan tangguh untuk melakukan operasi tim merah, dengan fokus pada sistem command and control (C2) dan alat serangan.

### Kampanye Serangan Multi-Tahap
Kampanye terkoordinasi yang meniru penyerang dunia nyata, melibatkan beberapa fase seperti pengintaian, eksploitasi, dan pergerakan lateral.

### Simulasi dan Emulasi Lawan
Simulasi realistis taktik yang digunakan oleh aktor ancaman tingkat lanjut untuk mengevaluasi langkah-langkah pertahanan organisasi dan mengidentifikasi area peningkatan.

## Penghindaran

### Bypass Deteksi Jaringan dan Endpoint
Menggunakan taktik untuk menghindari deteksi oleh sistem IDS/IPS dan solusi keamanan endpoint, melalui pengaburan malware atau mengeksploitasi celah deteksi.

### Anti-Forensik dan Manipulasi Bukti
Teknik yang dirancang untuk menggagalkan analisis forensik atau menghapus jejak bukti, penting dalam mempertahankan operasi rahasia.

### Teknik Obfuscation dan Enkripsi
Menutupi payload atau komunikasi melalui lapisan obfuscation atau enkripsi untuk menghindari langkah-langkah keamanan dan analisis.

### Penghindaran Honeypot dan Sandbox
Mendeteksi dan menghindari interaksi dengan sistem umpan seperti honeypot atau lingkungan sandbox yang bertujuan untuk menangkap aktivitas berbahaya.

## Pengembangan Malware

### Perancangan dan Pengiriman Payload Khusus
Menciptakan payload yang disesuaikan, sering kali menargetkan lingkungan tertentu atau melewati pertahanan yang ada, menekankan penyamaran dan fungsionalitas.

### Teknik Penghindaran dan Persistensi Malware
Merancang malware agar bertahan di sistem pasca-kompromi dan menghindari deteksi melalui obfuscation atau manipulasi pertahanan host.

### Pengembangan dan Manajemen Infrastruktur C2
Membangun dan mengelola infrastruktur command and control untuk mempertahankan otonomi atas sistem yang dikompromikan, penting bagi aktor ancaman persisten.

### Metode Bypass Anti-Malware
Upaya untuk melewati sistem deteksi malware otomatis menggunakan teknik penghindaran seperti manipulasi tanda tangan atau modifikasi perilaku.

## Perangkat Lunak

### Teknik Backdooring Perangkat Lunak
Memperkenalkan titik akses terselubung ke dalam kode perangkat lunak untuk akses tidak sah, sering kali melalui basis kode sumber terbuka atau milik yang dimanipulasi.

### Manipulasi Logika Aplikasi
Memanfaatkan kesalahan dalam logika bisnis atau aliran aplikasi untuk mengeksekusi tindakan yang tidak diinginkan atau secara tidak benar mengekstraksi data.

### Taktik Rekayasa Balik Lanjutan
Analisis mendalam terhadap eksekusi perangkat lunak menggunakan alat seperti IDA Pro atau Ghidra untuk mengekspos logika internal atau kerentanan dalam sistem tertutup.

### Injeksi Kode dan Hooking
Menyuntikkan kode berbahaya ke dalam proses yang berjalan atau mencegat alur kerja aplikasi melalui teknik hooking, sering kali digunakan untuk peningkatan hak istimewa.
