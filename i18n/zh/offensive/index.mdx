# 网络安全攻防路线图

## 网络

### 侦察和网络映射
网络侦察涉及收集目标网络的信息，以绘制其结构和设备的地图。技术包括被动信息收集、扫描和主动枚举网络组件。使用 Nmap 和 Wireshark 等工具识别开放端口、服务和潜在漏洞。使用类似以下命令：

```bash
nmap -sS -O -v target-ip
```

此命令执行 TCP SYN 扫描并检测目标操作系统。

### 横向移动策略
横向移动指攻击者在初次入侵后通过网络的渗透技术。这包括利用相邻网络段的漏洞和使用被盗凭据访问资源。攻击者可能使用 PsExec 或 PowerShell 等工具进行横向传播。

### 利用网络协议
这涉及识别网络协议（如 SMB、RDP 或 DNS）的弱点并利用它们执行未授权操作。对协议机制和异常创建的了解可以导致成功利用。

### 中间人攻击技术
中间人攻击涉及在不被检测的情况下拦截和修改双方之间的通信。技术可能包括 ARP 欺骗或 DNS 投毒。攻击者使用 Ettercap 和 MITMf 等工具获取和操纵流量。

### 网络流量分析与操控
流量分析涉及监控网络数据包，以识别模式或异常。攻击者可能会重新路由、丢弃或注入数据包以干扰或分析通信。Tcpdump 和 Scapy 等工具通常用于这些目的。

## Web 应用

### 高级注入攻击(SQLi, NoSQLi)
SQL 注入涉及将恶意 SQL 代码插入用户输入字段以操控或提取数据库数据。该原理扩展到 NoSQL 数据库即 NoSQLi。SQLmap 是用于自动 SQL 注入和数据库接管的常见工具：

```bash
sqlmap -u "http://target.com/search?q=test" --dbs
```

此命令尝试从目标 URL 提取数据库名称。

### 跨站脚本(XSS)和CSRF利用
XSS 允许攻击者将脚本注入到其他用户查看的网页中，从而实现会话劫持或数据盗窃。CSRF 诱惑用户在已认证的不同网站上执行不想要的操作。

### 不安全直接对象引用(IDOR)和BOLA
IDOR 漏洞允许攻击者通过操控对对象（如用户ID）的引用来访问数据。破坏的对象级授权(BOLA)涉及不正确的单个对象访问控制执行。

### Web Shell 技术与持久化
在被入侵的服务器上安装 web shell，以便持久访问和远程命令执行。web shell 是提供 HTTP 接口的简单脚本。

```php
<?php system($_GET['cmd']); ?>
```

这是简单的 PHP web shell 的例子。

### Web 应用防火墙规避
规避 WAF 涉及创建绕过基于签名检测的有效载荷。技术包括编码、混淆或攻击向量的分片。

## 移动应用

### 逆向工程移动应用
逆向工程涉及解密和分析移动应用代码以了解其逻辑并识别漏洞。APKTool 或 Hopper 之类的工具对于反汇编或反编译至关重要。

### 移动平台漏洞利用
每个平台（如 Android、iOS）都有特定漏洞。利用可能涉及利用系统特权或过时库。

### 拦截和操控移动通信
攻击者使用 Burp Suite 或 Frida 等工具拦截和操控移动应用传输的数据，尤其是当 SSL/TLS 实现较弱或配置错误时。

### 移动应用数据安全漏洞
针对移动应用使用的非安全数据存储或数据传输方法，在未充分加密情况下存储或通信敏感信息。

## 供应链

### 针对开发和交付管道
入侵开发管道可导致恶意代码的广泛分发。攻击者针对源代码库或 CI/CD 环境注入后门。

### 嵌入式系统安全测试
专注于识别嵌入设备固件或软件中的漏洞。测试通常包括硬件接口如 JTAG 或 UART。

### 供应链妥协技术
将恶意组件插入到产品的供应链中，在到达最终用户之前破坏完整性。著名实例包括硬件植入或篡改的更新文件。

### 第三方软件植入后门
在开发过程中，将隐藏的、未授权的代码引入第三方供应商提供的软件产品中，无论是有意为之还是通过漏洞完成。

## CI/CD

### 入侵 CI/CD 环境
未授权访问 CI/CD 环境赋予更改软件构建流程的权限，可能在部署前插入恶意代码。

### 侵入构建管道
攻击者拦截或修改构建产物，导致分发被入侵的软件。通常利用构建脚本或依赖中的漏洞。

### 产物篡改和完整性损害
未被检测的构建产物修改会破坏软件的预期功能。确保产物完整性至关重要，依赖于校验和或加密签名。

### CI/CD 工具漏洞利用
已知 CI/CD 工具的漏洞，如 Jenkins 或 GitLab，可被利用以获取未授权访问和代码更改。

## 云

### 云服务利用技术
利用云服务中的错误配置或漏洞来获得未授权的访问或云管理资源控制权。

### 云存储错误配置攻击
针对由于安全控制不足（如公共写入或读取权限）导致的云存储服务暴露，如 AWS S3 存储桶。

### 云身份和访问管理绕过
通过利用弱者 IAM 策略或过于宽松的角色，获取未授权云资产访问。

### 无服务器架构攻击向量
通过滥用事件触发或在功能执行上下文中注入恶意载荷，利用无服务器框架中的漏洞。

## 容器

### 容器逃逸技术
识别和利用允许恶意代码从容器环境突破到宿主系统的漏洞。

### 攻击 Kubernetes 集群
利用 Kubernetes 环境中的错误配置以获得未授权控制权或中断服务。

### 利用容器编排漏洞
针对编排工具（如 Docker Swarm 或 Kubernetes）中的漏洞来访问受限环境。

### 容器镜像中毒
通过注入恶意代码篡改容器镜像，影响部署这些镜像的系统。

## API

### API 终点枚举和侦察
识别 API 终点并绘制其结构，以确定可用资源和潜在漏洞。

### API 认证和授权绕过
利用允许绕过认证层或不正确的授权检查的漏洞。

### 速率限制和配额滥用
通过操控请求头或载荷绕过速率限制，以压垮 API 或提取比预期更多的数据。

### 利用 REST 和 SOAP API 漏洞
通过注入、反序列化攻击或利用详细错误信息暴露对特定 API 的攻击。

## 硬件

### 硬件接口利用
操控或访问硬件接口以提取敏感数据或修改设备操作。

### 硬件侧信道攻击
利用电磁发射、功耗分析或时序信息推断设备操作或检索敏感信息。

### 嵌入设备入侵
通过固件或外部通信接口中的漏洞获取未授权访问嵌入设备。

### 硬件固件逆向工程
分析硬件设备的固件通过逆向工程识别漏洞或了解设备操作。

## 无线

### 无线网络攻击（Wi-Fi、蓝牙）
利用弱 Wi-Fi 协议（WEP/WPA2）或蓝牙错误配置以拦截和操控无线通信。

### 搭建流氓接入点
部署未授权接入点以拦截无线通信，此技术常用于钓鱼或凭据盗窃。

### 窃听无线通信
监听无线网络上的未加密通信以捕获敏感信息如密码。

### 无线协议利用
利用诸如 Zigbee 或蓝牙低功耗（BLE）等协议的弱点以控制或监听设备通信。

## 物理安全攻击

### 社交工程与物理渗透
使用欺骗来绕过物理安全措施，涉及冒充或凭据伪造以获得未授权设施访问。

### 绕过物理访问控制
通过技术手段如锁撬或中继攻击绕过锁或生物认证系统等障碍。

### 物理层信息外泄
通过物理手段提取数据，如从暴露的硬盘中复制数据或截获电磁信号。

### 物理安全评估方法
基于模拟的技术评估物理安全设置中的漏洞，涉及设施的渗透测试。

## 加密

### 加密协议攻击
利用加密协议中的漏洞导致数据解密、认证绕过或冒充。

### 密码分析技术
技术采用数学和系统分析以妥协加密安全，包括破解密码。

### 密钥管理滥用
攻击不当的密钥管理实践导致未授权的密钥泄漏和数据解密。

### 破解对称和非对称加密
寻找弱点或利用实现缺陷以在无密钥持有下解密数据。示例包括对 AES 或 RSA 的攻击。

## 漏洞开发

### 识别漏洞利用机会
研究和发现可以被利用的软件或硬件中的漏洞。重点是未清洗输入或未检查操作中的入口点。

### 高级内存损坏技术
利用内存操作中的弱点实现任意代码执行，包括缓冲区溢出和重用后释放的漏洞。

### Shellcode 设计与传递
设计和注入 shellcode 作为执行目标系统上恶意指令的有效载荷。

### 漏洞框架的利用
使用 Metasploit 等工具协助漏洞开发和管理通过配置模块针对不同目标的攻击向量。

## 红队

### 红队参与的计划与执行
红队模拟高级持续性威胁（APT）以评估组织防御。规划侧重于实现特定战术、技术和程序（TTP）。

### 红队基础设施和工具设置
构建一个安全和有弹性的基础设施用于进行红队行动，注重指挥与控制（C2）系统和攻击工具。

### 多阶段攻击活动
模拟真实攻击者的协调行动，涉及多个阶段如侦察、利用和横向移动。

### 对手模拟与仿真
对先进威胁演员使用的战术进行现实模拟，以评估组织的防御措施并识别改进领域。

## 规避

### 网络和终端检测绕过
利用战术避免 IDS/IPS 和终端安全解决方案检测，通过恶意软件混淆或利用检测间隙。

### 反取证和证据篡改
设计来阻挠法证分析或磨损证据痕迹的技术，对于维持隐秘操作至关重要。

### 混淆与加密技术
通过层叠的混淆或加密掩盖有效载荷或通信，以规避安全措施和分析。

### 蜜罐和沙盒规避
检测并避免与旨在捕获恶意活动的诱饵系统如蜜罐或沙盒环境的交互。

## 恶意软件开发

### 自定义有效载荷设计与传递
创建针对特定环境或绕过现有防御的量身定制有效载荷，强调隐蔽性和功能性。

### 恶意软件规避和持久化技术
设计恶意软件以在妥协后持久存在于系统并通过混淆或主机防御操控避免检测。

### C2 基础设施开发与管理
构建和管理指挥与控制基础设施以保持对被入侵系统的自主权，对持续性威胁演员至关重要。

### 反恶意软件绕过方法
使用规避技术如签名操控或行为修改绕过自动恶意软件检测系统的努力。

## 软件

### 软件植入后门技术
在软件代码中引入秘密访问点以获得未授权访问，通常通过被操控的开源或专有代码库实现。

### 应用逻辑操控
利用业务逻辑或应用流程中的错误以执行未预期的动作或不正确地提取数据。

### 高级逆向工程策略
使用 IDA Pro 或 Ghidra 等工具对软件可执行文件进行深入分析以揭露内部逻辑或封闭源系统中的漏洞。

### 代码注入和钩挂
将恶意代码注入正在运行的进程或通过钩子技术拦截应用工作流程，通常用于权限提升。
