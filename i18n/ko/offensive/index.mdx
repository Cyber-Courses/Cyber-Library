# 공격 사이버 보안 로드맵

## 네트워크

### 정찰 및 네트워크 매핑
네트워크 정찰은 대상 네트워크에 대한 정보를 수집하여 구조와 장치의 지도를 만드는 과정입니다. 기술에는 수동 정보 수집, 스캔 및 네트워크 구성 요소의 적극적인 열거가 포함됩니다. Nmap 및 Wireshark와 같은 도구가 열려 있는 포트, 서비스 및 잠재적 취약점을 식별하는 데 사용됩니다. 다음과 같은 명령을 사용하십시오:

```bash
nmap -sS -O -v target-ip
```

이 명령은 TCP SYN 스캔을 수행하고 대상 운영 체제를 감지합니다.

### 횡적 이동 전략
횡적 이동은 초기 접근 후 네트워크를 통해 피벗하는 기술을 말합니다. 여기에는 인접 네트워크 세그먼트의 취약점을 악용하고 손상된 자격 증명을 사용하여 리소스에 접근하는 것이 포함됩니다. 공격자는 PsExec 또는 PowerShell과 같은 도구를 활용하여 횡적 전파를 수행할 수 있습니다.

### 네트워크 프로토콜 익스플로잇
이는 SMB, RDP, DNS와 같은 네트워크 프로토콜의 약점을 식별하고 이를 활용하여 무단 작업을 실행하는 것을 포함합니다. 프로토콜 메커니즘과 이상 현상 생성에 대한 지식은 성공적인 악용으로 이어질 수 있습니다.

### 중간자(MitM) 공격 기법
MitM 공격은 식별하지 못하고 두 당사자 간의 통신을 가로채고 변경하는 것을 포함합니다. 기법에는 ARP 스푸핑 또는 DNS 중독이 포함될 수 있습니다. 공격자는 트래픽을 캡처하고 조작하는 도구로 Ettercap 및 MITMf를 사용합니다.

### 네트워크 트래픽 분석 및 조작
트래픽 분석은 네트워크 데이터 패킷을 모니터링하여 패턴이나 이상 발생을 식별하는 과정입니다. 공격자는 통신을 파괴하거나 해부하기 위해 패킷을 재배치, 삭제 또는 주입할 수 있습니다. Tcpdump 및 Scapy와 같은 도구가 일반적으로 이러한 목적에 사용됩니다.

## 웹 애플리케이션

### 고급 인젝션 공격(SQLi, NoSQLi)
SQL 인젝션은 사용자 입력 필드에 악의적인 SQL 코드를 삽입하여 데이터베이스에서 데이터를 조작하거나 추출하는 것을 포함합니다. 이 원리는 NoSQL 데이터베이스에 NoSQLi로 확장됩니다. SQLmap은 SQL 인젝션과 데이터베이스 인수를 자동화하는 일반적인 도구입니다:

```bash
sqlmap -u "http://target.com/search?q=test" --dbs
```

이 명령은 대상 URL에서 데이터베이스 이름을 추출하려고 시도합니다.

### 크로스 사이트 스크립팅(XSS) 및 CSRF 악용
XSS는 공격자가 다른 사용자가 보는 웹 페이지에 스크립트를 삽입하여 세션 하이재킹이나 데이터 도난을 가능하게 합니다. CSRF는 사용자를 속여 그들이 인증된 다른 사이트에서의 원하지 않는 행동을 실행하게 합니다.

### 취약한 직접 객체 참조(IDOR) 및 BOLA
IDOR는 객체에 대한 참조(예: 사용자 ID)를 조작하여 데이터를 액세스할 수 있는 취약점을 포함합니다. Broken Object Level Authorization (BOLA)은 개별 객체에 대한 접근 제어가 올바르게 이루어지지 않는 것입니다.

### 웹 셸 전술 및 지속성
침입한 서버에 웹 셸을 설치하여 지속 가능한 접근과 원격 명령 실행을 가능하게 합니다. 웹 셸은 HTTP를 통해 인터페이스를 제공하는 간단한 스크립트입니다.

```php
<?php system($_GET['cmd']); ?>
```

간단한 PHP 웹 셸의 예입니다.

### 웹 애플리케이션 방화벽 회피
WAF를 피하는 것은 서명 기반 탐지를 우회하는 페이로드 제작을 포함합니다. 기법에는 인코딩, 눈속임 또는 공격 벡터의 단편화가 포함됩니다.

## 모바일 애플리케이션

### 모바일 애플리케이션 리버스 엔지니어링
리버스 엔지니어링은 모바일 앱의 코드를 해독하고 분석하여 그 로직을 이해하고 취약점을 식별하는 것입니다. APKTool 또는 Hopper와 같은 도구는 분해 또는 역분해에 필수적입니다.

### 모바일 플랫폼 취약점 익스플로잇
각 모바일 플랫폼(예: Android, iOS)에는 특정 취약점이 있습니다. 익스플로잇은 시스템 권한 또는 구식 라이브러리를 활용하는 것을 포함할 수 있습니다.

### 모바일 통신 가로채기 및 조작
공격자는 Burp Suite 또는 Frida와 같은 도구를 사용하여 모바일 애플리케이션에서 전송되는 데이터를 가로채고 조작하며, 특히 SSL/TLS 구현이 약하거나 잘못 구성된 경우에 유리합니다.

### 모바일 애플리케이션 데이터 보안 침해
민감한 정보를 적절한 암호화 없이 저장하거나 통신하는 모바일 앱이 사용하는 불안전한 데이터 저장 또는 데이터 전송 방법을 대상으로 합니다.

## 공급망

### 개발 및 전송 파이프라인 타격
개발 파이프라인을 침해하면 악성 코드의 광범위한 배포로 이어질 수 있습니다. 공격자는 소스 코드 리포지토리 또는 CI/CD 환경을 타깃으로 하여 백도어를 주입합니다.

### 임베디드 시스템 보안 테스트
펌웨어 또는 임베디드 장치의 소프트웨어에 있는 취약점을 식별하는 데 초점을 맞추십시오. 테스트는 종종 JTAG 또는 UART와 같은 하드웨어 인터페이스를 포함합니다.

### 공급망 침해 기법
제품의 공급망 내에 악성 구성 요소를 삽입하여 최종 사용자에게 도달하기 전에 무결성을 훼손합니다. 주목할만한 사례로는 하드웨어 임플란트 또는 변경된 업데이트 파일이 포함됩니다.

### 서드파티 소프트웨어 백도어
개발 중에 서드파티 공급자가 제공하는 소프트웨어 제품에 숨겨진 무단 코드를 도입하며, 이는 의도적이거나 익스플로잇을 통해 가능합니다.

## CI/CD

### CI/CD 환경 침해
CI/CD 환경에 대한 무단 접근은 배포 전에 소프트웨어 빌드 프로세스를 변경하여 악의적인 코드를 삽입할 수 있습니다.

### 빌드 파이프라인 침투
공격자는 빌드 아티팩트를 가로채거나 수정하여 손상된 소프트웨어의 배포로 이어집니다. 빌드 스크립트 또는 종속성의 취약점이 일반적으로 악용됩니다.

### 아티팩트 조작 및 무결성 손상
소프트웨어의 의도된 기능을 저해할 수 있는 빌드 아티팩트의 수정은 감지되지 않을 수 있습니다. 아티팩트의 무결성을 확인하는 것은 체크섬이나 암호화 서명에 의존합니다.

### CI/CD 도구 취약점 익스플로잇
Jenkins 또는 GitLab과 같은 CI/CD 도구의 알려진 취약점을 무단 접근 및 코드 변경에 활용할 수 있습니다.

## 클라우드

### 클라우드 서비스 악용 기법
클라우드 서비스의 잘못된 구성이나 취약점을 악용하여 클라우드 관리 리소스에 대한 무단 접근이나 제어를 획득합니다.

### 잘못 구성된 클라우드 저장소 공격
AWS S3 버킷과 같은 노출된 저장소 서비스를 표적으로 하여 공개 쓰기 또는 읽기 접근과 같은 불충분한 보안 제어를 악용합니다.

### 클라우드 ID 및 접근 관리 우회
약한 IAM 정책이나 과도하게 허용적인 역할을 악용하여 클라우드 자산에 무단 접근을 얻습니다.

### 서버리스 아키텍처 공격 벡터
서버리스 프레임워크의 취약점을 악용하여 이벤트 트리거를 남용하거나 기능 실행 컨텍스트에 악성 페이로드를 주입합니다.

## 컨테이너

### 컨테이너 이스케이프 기법
악의적인 코드가 컨테이너 환경에서 호스트 시스템으로 탈출하도록 허용하는 취약점을 식별하고 악용합니다.

### 쿠버네티스 클러스터 공격
쿠버네티스 환경 내 잘못된 구성을 악용하여 무단 제어를 획득하거나 서비스를 중단시킵니다.

### 컨테이너 오케스트레이션 취약점 익스플로잇
Docker Swarm 또는 Kubernetes와 같은 오케스트레이션 도구의 취약점을 악용하여 제한된 환경에 접근합니다.

### 컨테이너 이미지 중독
컨테이너 이미지에 악의적인 코드를 주입하여 이러한 이미지가 배포된 시스템에 영향을 미칩니다.

## API

### API 엔드포인트 열거 및 정찰
API 엔드포인트를 식별하고 구조를 매핑하여 가용 리소스와 잠재적 취약점을 확인합니다.

### API 인증 및 권한 부여 우회
인증 계층 또는 부적절한 권한 확인을 우회하는 취약점을 악용합니다.

### 제한 및 할당량 악용
요청 헤더나 페이로드를 조작하여 제한을 우회하여 API를 과부하시키거나 의도보다 더 많은 데이터를 추출합니다.

### REST 및 SOAP API 취약점 악용
주입, 역직렬화 공격 또는 자세한 오류 메시지로 인한 노출을 통해 특정 API를 타겟팅합니다.

## 하드웨어

### 하드웨어 인터페이스 악용
하드웨어 인터페이스를 조작하거나 접근하여 민감한 데이터를 추출하거나 장치 작동을 수정합니다.

### 하드웨어의 사이드 채널 공격
전자기 방사, 전력 소모 분석 또는 시간 정보를 활용하여 장치 작동을 추론하거나 민감한 정보를 검색합니다.

### 임베디드 장치 침입
펌웨어나 외부 통신 인터페이스의 취약점을 통해 임베디드 장치에 무단 접근합니다.

### 하드웨어 펌웨어 리버스 엔지니어링
하드웨어 장치의 펌웨어를 분석하여 취약점을 식별하거나 리버스 엔지니어링을 통해 장치 작동을 이해합니다.

## 무선

### 무선 네트워크 공격(Wi-Fi, 블루투스)
약한 Wi-Fi 프로토콜(WEP/WPA2) 또는 블루투스 구성 오류를 악용하여 무선 통신을 가로채고 조작합니다.

### 불량 액세스 포인트 배포
무선 통신을 가로채기 위해 승인되지 않은 액세스 포인트를 배포하며, 이는 피싱이나 자격 증명 도난에 자주 사용됩니다.

### 무선 통신 도청
무선 네트워크에서 암호화되지 않은 통신을 수신하여 비밀번호와 같은 민감한 정보를 캡처합니다.

### 무선 프로토콜 익스플로잇
Zigbee 또는 블루투스 저에너지(BLE)와 같은 프로토콜의 약점을 악용하여 장치 통신을 제어하거나 감시합니다.

## 물리적 보안 공격

### 사회공학 및 물리적 침투
사기 수법을 사용하여 물리적 보안 조치를 우회하며, 불법적인 접근을 얻기 위해 사칭이나 자격 증명 위조를 포함합니다.

### 물리적 접근 제어 우회
잠금 장치 또는 바이오메트릭 시스템과 같은 물리적 장벽을 우회하며, 잠금 픽킹이나 릴레이 공격과 같은 기법을 사용합니다.

### 물리적 계층 정보 유출
노출된 하드 드라이브에서 데이터를 복사하거나 전자기 신호를 가로채는 것과 같은 물리적 수단을 통해 데이터를 추출합니다.

### 물리적 보안 평가 방법
시설 침투 테스트를 포함하여 물리적 보안 설정의 취약점을 평가하기 위한 시뮬레이션 기반 기술입니다.

## 암호화

### 암호화 프로토콜 공격
암호화 프로토콜의 취약점을 악용하여 데이터 복호화, 인증 우회 또는 사칭으로 이어질 수 있습니다.

### 암호 해독 기법
수학적 및 체계적 분석을 사용하여 암호화를 절충하는 기법으로, 암호 해독을 포함합니다.

### 키 관리 남용
부적절한 키 관리 관행을 공격하여 무단 키 공개 및 데이터 복호화를 초래할 수 있습니다.

### 대칭 및 비대칭 암호화 해독
키 소유 없이 데이터를 복호화하도록 구현 결함을 찾아내거나 활용합니다. AES 또는 RSA에 대한 공격이 그 예입니다.

## 익스플로잇 개발

### 취약점 악용 기회 식별
소프트웨어 또는 하드웨어에서 악용할 수 있는 취약점을 연구하고 발견합니다. 검증되지 않은 입력이나 확인되지 않은 작업 내의 진입 지점에 집중합니다.

### 고급 메모리 손상 기법
버퍼 오버플로 및 사용 후 해제를 포함하여 임의 코드 실행을 달성하기 위해 메모리 작업의 약점을 활용합니다.

### 셸코드 제작 및 전달
목적 시스템에서 악의적인 명령을 실행하기 위해 페이로드로 셸코드를 설계하고 주입합니다.

### 익스플로잇 프레임워크 활용
Metasploit과 같은 도구를 사용하여 익스플로잇 개발을 지원하고 다양한 대상을 위한 구성 가능한 모듈을 통해 공격 벡터를 관리합니다.

## 레드 팀 활동

### 레드 팀 활동 계획 및 실행
레드 팀 활동은 조직 방어를 평가하기 위한 고급 지속적 위협(APT)을 시뮬레이션합니다. 계획은 특정 전술, 기술 및 절차(TTP)를 달성하는 데 중점을 둡니다.

### 레드 팀 인프라 및 도구 설정
명령 및 통제(C2) 시스템 및 공격 도구에 집중하여 레드 팀 운영을 수행하기 위한 안전하고 복원력 있는 인프라를 구축합니다.

### 다단계 공격 캠페인
동기화된 캠페인으로 다단계 공격을 수행하여 현실의 공격자를 흉내내어 다양한 단계를 포함하여 정찰, 악용 및 횡적 이동을 수행합니다.

### 적 모방 및 시뮬레이션
조직의 방어 조치를 평가하고 개선 영역을 식별하기 위해 고급 위협 행위자가 사용하는 전술을 현실적으로 시뮬레이션합니다.

## 회피

### 네트워크 및 엔드포인트 탐지 우회
악성 소프트웨어 난독화 또는 탐지 갭 악용을 통해 IDS/IPS 및 엔드포인트 보안 솔루션을 우회하는 전술을 활용합니다.

### 안티포렌식 및 증거 조작
포렌식 분석을 저지하거나 증거 경로를 침식하기 위한 기술로, 은밀한 작전을 유지하는 데 중요합니다.

### 난독화 및 암호화 기법
성능 및 분석을 회피하기 위해 페이로드 또는 통신을 난독화 또는 여러 계층의 암호화를 통해 가립니다.

### 허니팟 및 샌드박스 회피
활동을 포착하려는 허니팟 또는 샌드박스 환경과의 상호 작용을 감지하고 회피합니다.

## 악성 코드 개발

### 맞춤형 페이로드 설계 및 배달
특정 환경을 표적으로 하거나 기존 방어를 우회하는 맞춤형 페이로드를 설계하며, 은밀성과 기능성을 중시합니다.

### 악성 코드 회피 및 지속성 기법
악의적인 코드가 난독화 및 호스트 방어 조작을 통해 시스템에서 지속적으로 실행되고 탐지를 회피하도록 만듭니다.

### C2 인프라 개발 및 관리
지속적인 위협 행위자에게 중요하게 영향을 미치는 시스템에 대한 자율성을 유지하기 위해 명령 및 통제 인프라를 구축하고 관리합니다.

### 안티 악성 코드 우회 방법
서명 조작 또는 행동 수정과 같은 회피 기법을 사용하여 자동화된 악성 코드 탐지 시스템을 우회하기 위해 노력합니다.

## 소프트웨어

### 소프트웨어 백도어 기법
오프소스 또는 라이선스가 있는 코드를 조작하여 소프트웨어 코드에 부당한 접근을 초래하는 숨겨진 접근점을 삽입합니다.

### 애플리케이션 로직 조작
비즈니스 로직 또는 애플리케이션 흐름의 오류를 악용하여 의도하지 않은 행동을 실행하거나 데이터를 잘못 추출합니다.

### 고급 리버스 엔지니어링 기법
IDA Pro 또는 Ghidra와 같은 도구를 사용하여 닫힌 소스 시스템에서 내부 로직 또는 취약점을 노출하기 위해 소프트웨어 실행 파일에 대한 심층 분석을 수행합니다.

### 코드 주입 및 후킹
권한 상승을 위해 런타임 프로세스에 악성 코드를 삽입하거나 후크 기법을 통해 애플리케이션 워크플로우에 개입하는 것입니다.
