# 공격적 사이버 보안 로드맵

## 네트워크

### 정찰 및 네트워크 매핑
네트워크 정찰은 목표 네트워크에 대한 정보를 수집하여 네트워크의 구조와 장치에 대한 지도를 작성하는 과정을 포함합니다. 기술에는 수동 정보 수집, 스캐닝 및 네트워크 구성 요소의 적극적인 열거가 포함됩니다. Nmap 및 Wireshark와 같은 도구를 활용하여 열려 있는 포트, 서비스 및 잠재적 취약점을 식별합니다. 다음과 같은 명령을 사용하십시오:

```bash
nmap -sS -O -v target-ip
```

이 명령은 TCP SYN 스캔을 수행하고 대상 운영 체제를 탐지합니다.

### 횡적 이동 전략
횡적 이동은 공격자가 초기 접근 후 네트워크를 통해 축 이동을 수행하는 기술을 의미합니다. 이는 인접 네트워크 세그먼트의 취약점을 악용하고, 손상된 자격 증명을 사용하여 리소스에 접근하는 것을 포함합니다. 공격자는 PsExec 또는 PowerShell과 같은 도구를 사용하여 횡적으로 전파할 수 있습니다.

### 네트워크 프로토콜 악용
이는 SMB, RDP 또는 DNS와 같은 네트워크 프로토콜의 약점을 식별하고 이를 활용하여 무단 작업을 수행하는 것을 포함합니다. 프로토콜 메커니즘과 비정상 상태의 생성에 대한 지식을 사용하여 성공적인 악용을 실현할 수 있습니다.

### 중간자 공격 기술
중간자(MitM) 공격은 두 당사자 간의 통신을 가로채고 이를 변경하여 탐지 없이 조작하는 것을 포함합니다. 기술에는 ARP 스푸핑 또는 DNS 중독이 포함될 수 있습니다. 공격자는 트래픽을 캡처하고 조작하기 위해 Ettercap 및 MITMf와 같은 도구를 활용합니다.

### 네트워크 트래픽 분석 및 조작
트래픽 분석은 네트워크 데이터 패킷을 모니터링하여 패턴이나 이상을 식별하는 것을 포함합니다. 공격자는 통신을 방해하거나 해체하기 위해 패킷을 재경로 시키거나 폐기하거나 주입할 수 있습니다. Tcpdump 및 Scapy와 같은 도구는 이러한 목적에 일반적으로 사용됩니다.

## 웹 애플리케이션

### 고급 인젝션 공격 (SQLi, NoSQLi)
SQL 인젝션은 사용자 입력 필드에 악성 SQL 코드를 삽입하여 데이터베이스에서 데이터를 조작하거나 추출합니다. 이 원칙은 NoSQL 데이터베이스를 사용한 NoSQLi에게도 확장됩니다. SQLmap은 SQL 인젝션을 자동화하고 데이터베이스 점령을 시도하는 일반적인 도구입니다:

```bash
sqlmap -u "http://target.com/search?q=test" --dbs
```

이 명령은 대상 URL에서 데이터베이스 이름을 추출하려고 시도합니다.

### 크로스 사이트 스크립팅 (XSS) 및 CSRF 취약점 악용
XSS는 공격자가 다른 사용자가 보는 웹 페이지에 스크립트를 주입하여 세션 하이재킹 또는 데이터 절도를 가능하게 합니다. CSRF는 사용자가 인증된 다른 사이트에서 원하지 않는 작업을 실행하도록 유도합니다.

### 불안전한 직접 객체 참조 (IDOR) 및 BOLA
IDOR 취약점은 공격자가 객체(예: 사용자 ID)에 대한 참조를 조작하여 데이터에 접근할 수 있도록 합니다. 객체 수준 인가 파손(BOLA)은 개별 객체에 대한 접근 제어의 부적절한 강제를 포함합니다.

### 웹 셸 전술 및 지속성
손상된 서버에 웹 셸을 설치하여 지속적인 접근과 원격 명령 실행을 가능하게 합니다. 웹 셸은 HTTP를 통해 인터페이스를 제공하는 간단한 스크립트입니다.

```php
<?php system($_GET['cmd']); ?>
```

간단한 PHP 웹 셸의 예입니다.

### 웹 애플리케이션 방화벽 회피
WAF를 회피하는 것은 시그니처 기반 탐지를 우회하기 위한 페이로드를 조작하는 것을 포함합니다. 기술에는 인코딩, 난독화 또는 공격 벡터의 분할이 포함됩니다.

## 모바일 애플리케이션

### 모바일 애플리케이션 역설계
역설계는 모바일 앱의 코드를 해독하고 분석하여 그 논리를 이해하고 취약점을 식별하는 것을 포함합니다. APKTool 또는 Hopper와 같은 도구는 비조립이나 역컴파일에 필수적입니다.

### 모바일 플랫폼 취약점 악용
각 모바일 플랫폼(Android, iOS 등)은 특정한 취약점을 가지고 있습니다. 악용은 시스템 권한 또는 오래된 라이브러리를 활용할 수 있습니다.

### 모바일 통신 가로채기 및 조작
공격자는 Burp Suite 또는 Frida와 같은 도구를 사용하여 약한 SSL/TLS 구현 또는 잘못 구성된 데이터 전송을 가로채고 조작합니다.

### 모바일 애플리케이션 데이터 보안 침해
데이터가 충분한 암호화 없이 저장되거나 전달되는 불안전한 데이터 저장 또는 전송 방법을 타겟으로 하는 모바일 앱에서 민감한 정보를 얻습니다.

## 공급망

### 개발 및 전달 파이프라인의 표적화
개발 파이프라인을 손상시키면 악성 코드의 광범위한 배포로 이어질 수 있습니다. 공격자는 소스 코드 저장소 또는 CI/CD 환경을 타겟팅하여 백도어를 삽입합니다.

### 임베디드 시스템 보안 테스트
펌웨어 또는 임베디드 장치의 소프트웨어의 취약점을 식별하는 데 중점을 둡니다. 테스트는 종종 JTAG 또는 UART와 같은 하드웨어 인터페이스를 포함합니다.

### 공급망 손상 기술
제품의 공급망 내에서 악성 구성 요소를 삽입하여 최종 사용자에게 도달하기 전에 무결성을 방해합니다. 중요한 예는 하드웨어 임플란트 또는 조작된 업데이트 파일을 포함합니다.

### 제3자 소프트웨어 백도어링
개발 중에 제3자 공급업체가 공급한 소프트웨어 제품에 숨겨진, 무단 코드를 도입합니다. 이는 의도적이거나 취약점을 통해 발생할 수 있습니다.

## CI/CD

### CI/CD 환경 손상
CI/CD 환경에 대한 무단 접근은 소프트웨어 빌드 프로세스를 제어할 수 있는 권한을 부여하여 배포 전에 악성 코드를 삽입할 수 있게 합니다.

### 빌드 파이프라인 침투
공격자는 빌드 아티팩트를 가로채거나 수정하여 손상된 소프트웨어를 배포할 수 있습니다. 빌드 스크립트나 종속 항목의 취약점을 흔히 이용합니다.

### 아티팩트 변조 및 무결성 손상
수정된 빌드 아티팩트가 탐지되지 않으면 소프트웨어의 의도된 기능을 왜곡할 수 있습니다. 콘텐츠 무결성 보장은 체크섬 또는 암호화 서명에 의존합니다.

### CI/CD 도구 취약점 악용
Jenkins 또는 GitLab과 같은 CI/CD 도구의 알려진 취약점을 무단 접근 및 코드 변경에 활용할 수 있습니다.

## 클라우드

### 클라우드 서비스 악용 기술
클라우드 관리 리소스에 대한 무단 접근 또는 제어를 위해 잘못 구성되거나 취약성을 악용합니다.

### 잘못 구성된 클라우드 스토리지 공격
공개 쓰기 또는 읽기 접근과 같은 불충분한 보안 통제로 인해 노출된 AWS S3 버킷과 같은 저장 서비스를 대상으로 합니다.

### 클라우드 IAM 우회
약한 IAM 정책 또는 과도하게 허용적인 역할을 악용하여 클라우드 자산에 무단 접근을 얻습니다.

### 서버리스 아키텍처 공격 벡터
서버리스 프레임워크의 취약점을 이벤트 트리거를 남용하거나 함수 실행 컨텍스트에 악의적 페이로드를 주입하여 악용합니다.

## 컨테이너

### 컨테이너 탈출 기술
컨테이너 환경에서 호스트 시스템으로 악성 코드가 탈출할 수 있게 하는 취약점을 식별하고 악용합니다.

### 쿠버네티스 클러스터 공격
쿠버네티스 환경의 잘못된 구성을 악용하여 무단 제어를 얻거나 서비스를 방해합니다.

### 컨테이너 오케스트레이션 취약점 악용
Docker Swarm 또는 Kubernetes와 같은 오케스트레이션 도구의 취약점을 타겟팅하여 제한된 환경에 접근합니다.

### 컨테이너 이미지 중독
컨테이너 이미지에 악성 코드를 주입하여 이를 배포하는 시스템에 영향을 줍니다.

## API

### API 엔드포인트 열거 및 정찰
API 엔드포인트를 식별하고 구조를 매핑하여 사용 가능한 리소스 및 잠재적 취약점을 결정합니다.

### API 인증 및 인가 우회
인증 레이어를 우회하거나 부적절한 인가 확인을 악용하는 취약점을 이용합니다.

### 속도 제한 및 할당 남용
요청 헤더나 페이로드를 조작하여 API를 과부화시키거나 의도한 것보다 더 많은 데이터를 추출하는 방식으로 속도 제한을 우회합니다.

### REST 및 SOAP API 취약점 악용
주입, 직렬화 공격 또는 자세한 오류 메시지를 통한 노출을 통해 특정 API를 타겟팅합니다.

## 하드웨어

### 하드웨어 인터페이스 악용
민감한 데이터를 추출하거나 장치 작동을 변경하기 위해 하드웨어 인터페이스를 조작하거나 액세스합니다.

### 하드웨어에 대한 부채널 공격
전자기 방사, 전력 소비 분석 또는 타이밍 정보를 활용하여 장치 작동을 추론하거나 민감한 정보를 획득합니다.

### 임베디드 장치 침해
펌웨어 또는 외부 통신 인터페이스의 취약점을 통해 임베디드 장치에 무단 접근을 획득합니다.

### 하드웨어 펌웨어 역설계
장치 작동을 이해하거나 취약점을 식별하기 위해 하드웨어 장치의 펌웨어를 분석합니다.

## 무선

### 무선 네트워크 공격 (Wi-Fi, 블루투스)
약한 Wi-Fi 프로토콜(WEP/WPA2)이나 블루투스의 잘못된 구성을 악용하여 무선 통신을 가로채고 조작합니다.

### 악성 액세스 포인트 배포
무선 통신을 가로채기 위해 무단 액세스 포인트를 배포하는 기술이며, 이는 피싱이나 자격 증명 절도에 종종 사용됩니다.

### 무선 통신 도청
무선 네트워크에서 암호화되지 않은 통신을 청취하여 비밀번호와 같은 민감한 정보를 캡처합니다.

### 무선 프로토콜 악용
Zigbee 또는 Bluetooth 저에너지(BLE)의 프로토콜의 취약점을 악용하여 장치 통신을 제어하거나 스파이합니다.

## 물리 보안 공격

### 사회 공학 및 물리적 침투
속임수를 사용하여 물리적 보안 장치를 우회하는 것으로, 허위 신분을 통해 시설에 무단 접근을 획득합니다.

### 물리적 접근 제어 우회
잠금 장치나 생체 인식 시스템과 같은 장벽을 피킹 또는 릴레이 공격을 통해 우회합니다.

### 물리적 계층 정보 탈취
노출된 하드 드라이브에서 데이터 복사 또는 전자기 신호를 가로채는 것과 같은 물리적 방법을 통해 데이터를 탈취합니다.

### 물리 보안 평가 방법
침투 테스트를 포함하여 건물의 물리적 보안 설정에서 취약점을 평가하기 위한 시뮬레이션 기반 기술입니다.

## 암호학적

### 암호화 프로토콜 공격
데이터 복호화, 인증 우회 또는 사칭을 초래하는 암호화 프로토콜의 취약점을 악용합니다.

### 암호 해독 기술
수학적 및 체계 분석을 사용하는 암호 해독 기술은 암호 보안을 손상시키기 위한 것입니다. 여기에는 암호 해독이 포함됩니다.

### 키 관리 남용
무단 키 노출과 데이터 복호화를 초래할 수 있는 부적절한 키 관리 관행을 공격합니다.

### 대칭 및 비대칭 암호 해독
AES 또는 RSA에 대한 공격 예제가 포함된, 데이터 암호 없이 데이터 해독을 위한 취약점과 구현 결함을 발견합니다.

## Exploit 개발

### 취약점 악용 기회 식별
악용할 수 있는 소프트웨어나 하드웨어의 취약점을 연구하고 발견합니다. 미확인 입력이나 확인되지 않은 작업 내의 진입점을 중점적으로 연구합니다.

### 고급 메모리 손상 기술
제한된 범위를 벗어난 읽기/쓰기 및 사용 후 해제 취약점과 같은 약점을 활용하여 임의 코드 실행을 실현합니다.

### 쉘코드 제작 및 전송
대상 시스템에서 악성 명령을 실행하도록 설계된 쉘코드를 페이로드로 작성하고 주입합니다.

### Exploit 프레임워크 활용
Metasploit과 같은 도구를 사용하여 개발을 돕고, 가변 목표를 위한 구성 가능한 모듈을 통해 공격 벡터를 관리합니다.

## Red Teaming

### Red Team 참여 계획 및 실행
Red Teaming은 APT(Advanced Persistent Threat)를 시뮬레이션하여 조직의 방어체계를 평가하는 것이다. 계획은 특정 TTP(전술, 기법 및 절차)를 달성하기 위한 것입니다.

### Red Team 인프라 및 도구 설정
Red Team 활동을 위한 명령 및 제어(C2) 시스템과 공격 도구에 중점을 두고 보안적이고 탄력적인 인프라를 구축합니다.

### 다단계 공격 캠페인
관찰된 단계가 포함된 실세계 공격자와 유사한 다단계 공격 캠페인에는 정찰, 악용 및 횡적 이동이 포함됩니다.

### 적대적 시뮬레이션 및 모의 실험
조직의 방어 체계를 평가하고 개선 분야를 식별하기 위해 고급 위협 행위자가 사용하는 전술을 현실적으로 시뮬레이션합니다.

## 회피

### 네트워크 및 엔드포인트 탐지 우회
악성코드 난독화 또는 탐지 격차 악용을 통해 IDS/IPS 및 엔드포인트 보안 솔루션 탐지를 피하기 위한 전술을 활용합니다.

### 반범죄 과학 및 증거 조작
은밀한 운영을 유지하는 데 중요하며 법의학 분석을 방해하거나 증거 추적을 침식하기 위한 기술을 설계합니다.

### 난독화 및 암호화 기술
보안 조치 및 분석을 회피하기 위해 페이로드 또는 통신을 난독화 또는 암호화 레이어로 마스킹합니다.

### 허니팟 및 샌드박스 회피
악의적인 활동을 포착하기 위한 허니팟이나 샌드박스 환경과의 상호작용을 감지하고 피하십시오.

## 악성코드 개발

### 맞춤형 페이로드 디자인 및 전달
특정 환경을 겨냥하거나 기존 방어 체계를 우회하도록 맞춤형 페이로드를 생성하여 은밀함과 기능성을 중요시합니다.

### 악성코드 은폐 및 지속성 기술
손상 후 시스템에 지속되기 위해 악성코드를 제작하고, 호스트 방어를 난독화하거나 조작하여 탐지를 회피합니다.

### C2 인프라 개발 및 관리
손상된 시스템에 대한 자율성을 유지하기 위해 명령 및 제어 인프라를 구축하고 관리하는 것은 지속적인 위협 행위자에게 중요합니다.

### 안티멀웨어 회피 방법
서명 조작이나 동작 변경과 같은 회피 기술을 사용하여 자동화된 악성코드 탐지 시스템을 회피하기 위한 노력입니다.

## 소프트웨어

### 소프트웨어 Backdooring 기술
종종 오픈소스 또는 독점 코드 베이스를 조작하여 소프트웨어 코드에 모호한 접근 포인트를 도입하여 무단 접근을 허용합니다.

### 애플리케이션 논리 조작
비즈니스 논리나 애플리케이션 흐름에서 오류를 악용하여 의도하지 않은 행동을 수행하거나 부적절하게 데이터를 추출합니다.

### 고급 역설계 전술
IDA Pro 또는 Ghidra와 같은 도구를 사용하여 소프트웨어 실행 파일을 깊이 분석하여 폐쇄형 시스템의 내부 논리 또는 취약점을 드러냅니다.

### 코드 주입 및 후킹
런타임 프로세스에 악성코드를 주입하거나 후킹 기술을 통해 애플리케이션 워크플로우를 가로채고 사용하는 것으로, 이는 종종 권한 상승에 사용됩니다.
